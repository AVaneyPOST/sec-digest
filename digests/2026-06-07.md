## General high-severity

- **Miasma worm reaches Microsoft — 73 Azure / Microsoft / Azure-Samples / MicrosoftDocs GitHub repos disabled in ~105 seconds.** Event date 2026-06-05; broad press cycle 2026-06-06. A malicious commit was pushed to `Azure/durabletask` using a previously-compromised contributor account; GitHub's automated worm-detection swept four Microsoft organisations in two tight waves. The payload is **not in the code path** — it's a workspace-level config file (`.cursor/`, `.vscode/`, Claude Code / Gemini CLI workspace defs) that triggers when a developer opens the repository in an AI-coding agent or modern IDE and silently runs a credential harvester against SSH keys, CLI configs, browser data, wallets, and CI/CD runner secrets. Miasma is a TeamPCP-released variant of the Mini-Shai-Hulud worm; first hit @redhat-cloud-services on 2026-06-01 (96 malicious versions across 32 packages in two batches), spread via Phantom-Gyp `binding.gyp` lifecycle abuse, now in Microsoft's tree. **Action:** Treat any developer workstation that opened a Microsoft Azure GitHub repo on 2026-06-05 in Cursor, VS Code (with copilot/chat), Claude Code, or Gemini CLI as potentially credential-compromised; rotate cloud and registry tokens from that host, walk GitHub Actions runner secrets, and block the workspace-config attack pattern at IDE policy. ## Worth watching

- **Verizon VoLTE IMS SIP missing IPsec integrity (CVE-2026-10629 / VU#615987).** Published by CERT/CC on 2026-06-02 — outside today's strict 24h window but still the most material telecom item open this week. Verizon's VoLTE deployment is observed transmitting SIP REGISTER, INVITE, MESSAGE, BYE, and UPDATE without ESP encapsulation and without Security-Client / Security-Server / Security-Verify headers, contradicting 3GPP TS 33.203 and GSMA IR.92. On-path attackers (compromised femto, malicious IMS intermediary, fronthaul interposer) can intercept, replay, or inject SIP signalling — enabling call hijack, SMS-over-IMS spoofing, BYE/CANCEL DoS, and manipulation of emergency-call routing. Verizon initially engaged with CERT/CC and then stopped participating in coordination; Apple iOS 26.5 carrier bundle (2026-05-11) ships IMS IPsec settings but there is no evidence the network is enforcing them. No update in the 24h window.
