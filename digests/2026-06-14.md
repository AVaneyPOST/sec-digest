## General high-severity

- **Arch Linux AUR — "Atomic Arch" supply-chain compromise expands.** First-wave 408 packages (June 11 per Sonatype) escalated to 1,500+ in second-wave reporting through June 12–13 (PrivacyGuides). Attackers adopted orphaned AUR packages, modified PKGBUILDs to pull `atomic-lockfile` and `js-digest` npm payloads delivering an eBPF-disguised rootkit and an infostealer targeting browser creds, SSH keys, env vars, and wallet data. Official Arch repos unaffected. Practitioner takeaway: any analyst laptop running AUR-sourced tooling needs creds rotated and host audited. [BleepingComputer](https://www.bleepingcomputer.com/news/security/over-400-arch-linux-packages-compromised-to-push-rootkit-infostealer/) · [The Hacker News](https://thehackernews.com/2026/06/over-400-arch-linux-aur-packages.html) · [Latest Hacking News (2026-06-13)](https://latesthackingnews.com/2026/06/13/aur-supply-chain-attack-arch-packages-backdoored/) · severity **HIGH**.

- **Splunk Enterprise CVE-2026-20253 — unauth RCE chain via PostgreSQL Sidecar.** CVSS 9.8. The `/v1/postgres/recovery/backup` and `/v1/postgres/recovery/restore` endpoints lack auth and are reachable through Splunk's main web app via internal proxy. Splunk Enterprise 10.x affected; AWS deployments are vulnerable by default. Patched in 10.4.0 / 10.2.4 / 10.0.7 / 9.4.12 / 9.3.13. Vendor advisory dated 2026-06-10; broad write-ups continued through 2026-06-13. [Splunk advisory SVD-2026-0610](https://advisory.splunk.com/advisories/SVD-2026-0610) · [GBHackers analysis](https://gbhackers.com/critical-splunk-enterprise-pre-auth/) · severity **CRITICAL**.

- **GreatXML — unpatched BitLocker bypass.** Researcher "Nightmare Eclipse" dropped a WinRE-based BitLocker bypass with physical access on hosts that have previously run a Defender Offline scan (drops `unattend.xml` to the recovery partition, gets SYSTEM shell over the encrypted volume). No CVE, no patch as of today. Microsoft has not commented publicly. Disclosure 2026-06-10, mainstream reporting through 2026-06-12. [SecurityWeek](https://www.securityweek.com/greatxml-zero-day-exploit-bypasses-bitlocker/) · [The Register — Nightmare Eclipse drop](https://www.theregister.com/security/2026/06/11/nightmare-eclipse-drops-claimed-bitlocker-bypass-for-microsoft-windows/5254371) · severity **NOTABLE** (requires physical access; high impact on field/stolen-laptop scenarios).

Demonstrates the gadget-chain working against real targets; useful tell when scoping client-side PeopleSoft estate. [BleepingComputer](https://www.bleepingcomputer.com/news/security/nottingham-university-data-breach-affects-over-450-000-students/) · [The Register](https://www.theregister.com/cyber-crime/2026/06/11/shinyhunters-raids-nottingham-uni-for-student-alumni-data/5253961).

Reported 2026-06-11; matters for telco IR teams because flat OSS/management LANs and SS7/Diameter signalling-adjacent server segments are exactly the kind of targets worm-mode hits hardest. Watch for an out-of-band fix; an SD-WAN compromise is operator-LAN-equivalent.
