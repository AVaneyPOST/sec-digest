## General high-severity
- **wp2shell — CVE IDs assigned, mechanism fully public, working PoC dropped (18 Jul)** — [CRITICAL] The two-CVE chain now formalised: CVE-2026-63030 (REST API batch-route confusion) + CVE-2026-60137 (SQL injection in `WP_Query author__not_in`). Rapid7's advisory (18 Jul) confirms the public PoC exercises the SQLi to read the database including WordPress admin password hashes — the final jump to code execution (crack hash, upload plugin) is not in the public exploit but is trivially derivable from the mechanism. Cloudflare's telemetry-side note: the vulnerable code path is only reachable when a persistent object cache (Redis / Memcached) is *not* in use — sites running WP object caching are protected in the interim. Affected: WP 6.9.0-6.9.4 and 7.0.0-7.0.1. Fixed: 6.9.5 and 7.0.2 (17 Jul, forced auto-update). Rapid7's blog notes no confirmed in-the-wild mass exploitation as of 18 Jul, but historical patch-diff → exploit lag on WP core RCEs of this shape has been days, not weeks.
