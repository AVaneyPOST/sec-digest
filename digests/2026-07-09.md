## Telecom
- **Ubiquiti UniFi Connect CVE-2026-50746 — CVSS 10.0, unauthenticated command injection** — [CRITICAL] Direct hit for last-mile ISPs and small carriers who standardised on UniFi. Root cause is CWE-284 (Improper Access Control): the command-execution pathway is reachable without going through an auth gate; network / low complexity / no user interaction. Fix: UniFi Connect ≥ 3.4.20. Reports estimate ~100,000 endpoints in scope. Sources: [BleepingComputer](https://www.bleepingcomputer.com/news/security/ubiquiti-warns-of-new-max-severity-unifi-os-vulnerability/), [SOCRadar advisory writeup](https://socradar.io/blog/ubiquiti-cve-2026-50746-unifi-connect/), [SecurityAffairs](https://securityaffairs.com/194978/security/ubiquiti-patches-critical-unifi-os-flaws-allowing-command-injection-and-privilege-escalation.html), [TechTimes](https://www.techtimes.com/articles/319919/20260708/unifi-cvss-100-flaw-exposes-100000-endpoints-unauthenticated-takeover.htm) (08 Jul 2026).
- **KDDI email-platform breach — continued coverage** — [NOTABLE] Coverage on 08 Jul revised the exposure envelope on the shared ISP email plane to "up to 14.2 million email logins" across KDDI and five other ISPs. No new root-cause disclosure since the 06 Jul MPHPT regulator report. Watch for the report contents to leak into press this week. Accenture publicly acknowledged after initially denying, saying it was "isolated" and "remediated at the source" — with no comment on scope, exfiltration path, or client impact. Sources: [Help Net Security](https://www.helpnetsecurity.com/2026/07/08/accenture-data-breach-2026/), [BleepingComputer](https://www.bleepingcomputer.com/news/security/accenture-confirms-breach-after-hacker-offers-stolen-data-for-sale/), [SecurityWeek](https://www.securityweek.com/accenture-confirms-data-breach-after-hacker-claims-source-code-theft/), [Cybersecurity Dive](https://www.cybersecuritydive.com/news/accenture-data-breach-access-keys-source-code/824694/), [SecurityAffairs](https://securityaffairs.com/194962/data-breach/a-hacker-claims-35-gb-of-accenture-source-code-the-company-discloses-the-data-breach.html).

## Worth watching
- **Bluekit Phishing-as-a-Service (Netcraft / Varonis / CloudSEK / Security Boulevard, coverage on 07–08 Jul)** — MFA-bypass PhaaS built on Browser-in-the-Middle rather than adversary-in-the-middle: the victim's browser sees a live rrweb DOM-stream of a real Microsoft login page hosted in the operator's browser, so credential/token capture happens in the attacker's session. Netcraft counted ~70 live hostnames in a single week (26 Jun). Phase-1 anti-analysis includes randomised CSS filter to defeat pixel-hash screenshotting, custom CAPTCHA, 1MB+ obfuscated JS rotated periodically, WebRTC-based IP mismatch detection. Detection posture that only checks TLS/MFA will not catch this — need session-level behavioural signals. Sources: [Netcraft](https://www.netcraft.com/blog/bluekit-phishing-as-a-service-threat), [Varonis](https://www.varonis.com/blog/bluekit), [Security Boulevard](https://securityboulevard.com/2026/07/thwarting-bluekit-phishing-as-a-service-attacks-on-microsoft-authentication/), [CloudSEK](https://www.cloudsek.com/blog/bluekit-phishing-as-a-service-phaas), [Cybersecurity News](https://cybersecuritynews.com/bluekit-paas-bypasses-mfa/).
- **DHS HSIN breach — investigation ongoing (surfaced 02 Jul, coverage into 08 Jul)** — Threat actor targeted HSIN plus a SharePoint collaboration system between late May and early June; unclassified but sensitive interagency traffic. Congressional briefings were requested; no attribution yet. Watch for a follow-on SharePoint CVE (CVE-2026-45659 is already on KEV) tie-back. Sources: [BleepingComputer](https://www.bleepingcomputer.com/news/security/dhs-confirms-hackers-breached-hsin-info-sharing-platform/), [Nextgov/FCW](https://www.nextgov.com/cybersecurity/2026/06/hackers-breached-dhs-information-sharing-network-people-familiar-say/414534/), [TechCrunch](https://techcrunch.com/2026/07/02/us-government-says-it-got-hacked-again/).
- **Salt Typhoon — still active inside US telcos (background)** — Now credited with compromises in 80+ countries per multiple 2026 backgrounders; February 2026 CISA supplementary advisory on Volt Typhoon still governs comms-sector hunt guidance. Nothing new in-window today, but any operator running unpatched Cisco IOS-XE (CVE-2023-20198) on carrier edge should treat that as an active compromise, not a future risk. Source: [Salt Typhoon backgrounder / Congress.gov IF12798](https://www.congress.gov/crs-product/IF12798), [Wikipedia summary](https://en.wikipedia.org/wiki/Salt_Typhoon).
- **CitrixBleed / NetScaler CVE-2026-3055 (SAML IDP memory over-read, CVSS 9.8)** — Not new this week, but confirmed 08 Jul roundup shows continuing large-scale exploitation. If you use NetScaler as your SAML IDP, this is the item to confirm patched. Source: [Threat-Modeling.com writeup](https://threat-modeling.com/citrix-netscaler-saml-idp-cve-2026-3055/), [Fortinet Community outbreak alert](https://community.fortinet.com/fortirecon-48/outbreak-alert-citrix-netscaler-memory-overread-vulnerability-227772).
