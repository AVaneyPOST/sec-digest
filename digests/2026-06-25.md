## Telecom

- **Cisco Unified Communications Manager — CVE-2026-20230 actively exploited** *(24 Jun 2026, HIGH/CRITICAL)*
  Originally disclosed 3 Jun as an unauth SSRF in the WebDialer service that can escalate to root. Over the weekend of 21–22 Jun, Defused honeypots observed `file://` payloads being used to write files; by 24 Jun automated Tor-sourced sweeps were dropping a rogue Apache Axis service → first-stage JSP file-writer → second-stage command-execution shell. WebDialer is off by default, but is widely enabled in operator and enterprise voice estates. Affected: 14.x < 14SU6, 15.x < 15SU5. Telco angle: this is voice core / contact-centre infrastructure; SSRF on UCM can pivot into the IMS/SBC adjacency. Patches available — *don't rely on patching alone, attackers may already have webshells*; audit `tomcat/webapps`, look for new admin accounts, and disable WebDialer where unused. [Help Net Security: https://www.helpnetsecurity.com/2026/06/24/cisco-unified-cm-flaw-exploited-to-drop-webshells-cve-2026-20230/] [SecurityWeek: https://www.securityweek.com/hackers-exploiting-cisco-unified-cm-vulnerability/] [The Hacker News: https://thehackernews.com/2026/06/cisco-patches-cve-2026-20230-in-unified.html]

- **Deutsche Bahn GSM-R nationwide outage** *(23 Jun ~21:59 → 24 Jun ~00:30 local, NOTABLE)*
  Every train in Germany stopped for ~2.5 hours after a failure on the GSM-R cab-radio network used for train-to-control-centre signalling. DB attributes the outage to a "scheduled swap of a technical component"; an unconfirmed DDoS rumour circulated in early reporting. Even if benign, this is the cleanest recent demonstration of a single-protocol/legacy radio dependency taking out a national rail network — and is being used in 5G-Americas / FRMCS migration discussions to push operators off GSM-R. Worth tracking the regulator post-mortem. [The Record: https://therecord.media/deutsche-bahn-railroad-gsmr-outage] [PBS: https://www.pbs.org/newshour/world/trains-across-germany-halted-for-hours-by-communication-system-outage]

- **Ericsson Packet Core Gateway — three DoS CVEs (CVE-2026-25657/-25658/-25659)** *(June 2026 PSIRT bulletin, NOTABLE)*
  Three improper-handling bugs in PCG: a continuous stream of malformed messages causes service degradation / crash-and-recover. Fixed in PCG 1.30. Self-clearing once the attacker stops, so impact is interconnect availability rather than persistent compromise — still relevant for anyone exposing PCG signalling/user-plane termination to roaming peers. Exact publish date inside the 24h window not confirmed from the snippet I could read; included here so operators have it on the radar. [Ericsson PSIRT (unreachable from this run; see "Notes"): https://www.ericsson.com/en/about-us/security/psirt/security-bulletin-packetcoregateway-june-2026]

## General high-severity

- **CISA KEV additions, 23 Jun 2026 — Lantronix + Ubiquiti** *(CRITICAL)*
  - **CVE-2025-67038** — Lantronix EDS5000 serial-to-Ethernet servers. CVSS 9.8. Username is concatenated unsanitised into the HTTP RPC failed-login log-write shell command → unauth OS command injection as root. Fixed in 2.2.0.0R1. EDS5000 is widely deployed in industrial / utility / building-control networks; many will sit behind little or no edge filtering.
  - **CVE-2026-34908 / -34909 / -34910** — Ubiquiti UniFi OS, all CVSS 10. Improper access control + path traversal + improper input validation chain to unauth remote command execution. Fixed in UniFi OS Server 5.0.8. Public reports describe attackers creating rogue `John Sim` admin accounts during automated sweeps — i.e. mass opportunistic exploitation already in progress.
  - FCEB remediation deadline: **26 Jun 2026**.
  [CISA alert: https://www.cisa.gov/news-events/alerts/2026/06/23/cisa-adds-four-known-exploited-vulnerabilities-catalog] [BleepingComputer: https://www.bleepingcomputer.com/news/security/cisa-warns-of-max-severity-ubiquiti-flaws-exploited-in-attacks/] [The Hacker News: https://thehackernews.com/2026/06/cisa-warns-critical-lantronix-eds5000.html]

- **"Cordyceps" — systemic GitHub Actions / CI-CD exploitation class** *(disclosed ~24 Jun, HIGH)*
  Novee Security's research describes a family of GitHub Actions workflow weaknesses (command injection via PR title/body, broken auth, artifact poisoning, privilege escalation across low- → high-privilege workflows) that an unauthenticated user can trigger with a single malicious comment or PR. Of ~30k high-impact OSS repos scanned, 654 were flagged, 300+ verified exploitable end-to-end, including repos at Microsoft, Google, Apache, and Cloudflare. The blast radius is "compromise the published artifact" → classic xz/SolarWinds-shape supply chain. [Novee blog: https://novee.security/blog/cordyceps/] [Dark Reading: https://www.darkreading.com/application-security/cordyceps-malicious-pull-requests-developer-workflows] [The Hacker News: https://thehackernews.com/2026/06/cordyceps-cicd-flaws-expose-300-github.html]

  - **CVE-2026-56274** Flowise — CVSS 9.9, RCE.
  - **CVE-2026-53753** Crawl4AI — CVSS 9.8, RCE.
