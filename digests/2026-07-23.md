## General high-severity

- **CVE-2026-50522 — Microsoft SharePoint Server pre-auth deserialization RCE (CVSS 9.8), active exploitation, machine-key theft.** The vulnerable path is SharePoint's WS-Federation sign-in endpoint (`/_trust/default.aspx`): a forged `SecurityContextToken` carries a malicious `.NET BinaryFormatter` payload in the cookie field; the affected deserialization path yields code execution as the service account, and — critically — a single successful request is enough to lift the SharePoint server's ASP.NET machine keys. With those keys an attacker can mint valid authentication tokens and retain access even after the server is patched, which is why every advisor is telling operators to **rotate machine keys and any potentially exposed credentials after patching**, not just apply the July fix. Affects SharePoint Enterprise Server 2016, SharePoint Server 2019, and SharePoint Server Subscription Edition. Public PoC surfaced 20 Jul; watchTowr's global honeypot network recorded successful exploitation attempts within hours; wide coverage and additional detail broke on 22 Jul. Multiple secondary trackers report the CVE was added to the CISA KEV catalogue on 22 Jul, though the direct CISA KEV alert page returned 403 to automated fetches — treat that as reported-not-primary-verified. [Help Net Security](https://www.helpnetsecurity.com/2026/07/22/sharepoint-cve-2026-50522-exploited/), [BleepingComputer](https://www.bleepingcomputer.com/news/security/critical-sharepoint-rce-flaw-exploited-to-steal-machine-keys/), [The Hacker News](https://thehackernews.com/2026/07/critical-sharepoint-rce-cve-2026-50522.html), [SecurityAffairs](https://securityaffairs.com/195760/security/public-poc-triggers-active-exploitation-of-critical-sharepoint-rce-vulnerability-cve-2026-50522.html), [SocRadar](https://socradar.io/blog/cve-2026-50522-poc-fuels-sharepoint-attacks/). Date: 2026-07-22 (broad coverage day). [CRITICAL]

- **CISA joint advisory AA26-097A updated (22 Jul 2026): Iranian-affiliated actors targeting PLCs, scope broadened.** CISA, FBI, NSA, EPA, DOE, USCYBERCOM CNMF, and Treasury refreshed the April advisory with (a) new detection guidance for malicious changes in *reusable code modules* inside Rockwell Automation PLC programs — a subtler tradecraft evolution than the earlier direct-download-of-project-file activity — and (b) an expanded manufacturer scope: Schneider Electric, Siemens, and "potentially other branded/manufactured PLCs" are now called out alongside Rockwell. The victim sectors remain Government Services & Facilities, Water & Wastewater, and Energy. No new named zero-day, but the scope expansion is meaningful for ICS-adjacent telecom operators (backhaul into water/energy sites) who may have exposure via shared OT/IT boundaries. ## Worth watching

- **SharePoint post-patch cleanup is the story of the next fortnight, not the patch.** With CVE-2026-50522 exploitation focused on machine-key theft, and prior SharePoint CVEs from the same wave (CVE-2026-58644, CVE-2026-56164) also actively exploited earlier in July, on-prem SharePoint estates need a coordinated "patch → rotate machine keys → hunt for forged tokens" workstream. Every operator that patched-without-rotating may still be compromised. This includes any telecom back-office SharePoint that hosts intra-company documents.
- **Iranian PLC advisory scope expansion.** Rockwell → Schneider/Siemens broadening tells you Iranian actors have generalised their tooling beyond one vendor. For telecom operators the read-across is anything running Schneider or Siemens PLCs inside cell-site power/HVAC controllers or long-haul microwave hut monitoring — historically an under-attended attack surface.
- **APT42 messaging-plane grooming.** Long-dwell WhatsApp/Telegram conversations before payload delivery are hard to detect at the endpoint. Telco security teams whose staff use consumer messaging apps for work — including anyone with signalling-firewall or SS7-monitoring access — should treat this as an insider-recruitment pattern, not just a phishing pattern, and remind privileged users about family-member pivots.
- **Second-order Salt Typhoon coverage.** No fresh named-victim disclosures in the 24h window; the campaign remains an ongoing baseline concern. Telco IR teams should keep hunting for the LotL indicators published in earlier joint advisories rather than waiting on new disclosures.
