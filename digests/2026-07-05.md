## Worth watching

- **SharePoint CVE-2026-45659 KEV deadline reached (2026-07-04).** Untrusted-data
  deserialization RCE, CVSS 8.8, Site-Member-auth required. FCEB deadline under
  BOD 26-04 was today. Storm-2603 / Warlock exploitation ongoing; same
  exploit-chain family that surfaces in Orange Belgium / Colt / SmarterTools
  reporting. Monday will show whether laggards get compromised or whether
  the deadline actually shifted the exposure curve. Practically: audit any
  on-prem SharePoint that sits adjacent to subscriber, CDR, or interconnect
  data.
  [The Hacker News](https://thehackernews.com/2026/07/sharepoint-rce-cve-2026-45659-added-to.html)
  · [SecurityWeek](https://www.securityweek.com/cisa-warns-of-actively-exploited-microsoft-sharepoint-vulnerability/)
  · KEV entry 2026-07-01, deadline 2026-07-04

- **Armored Likho / BusySnake Stealer (Kaspersky Securelist, 2026-07-03).**
  Previously undocumented APT (overlaps with BI.ZONE's Eagle Werewolf cluster,
  active since 2023). Spear-phishing (NSIS droppers + `.lnk` chains) → Python
  infostealer → browser creds, session cookies, OTPs, Telegram sessions,
  wallet files, reverse SSH tunnel. Notable for two things: (a) targeting the
  electric-power sector alongside government agencies in Russia, Kazakhstan
  and Brazil, and (b) first-stage loader code that shows LLM authorship
  fingerprints — verbose inline comments, emoji bullet-points in source,
  redundant blocks. Detection engineers: those loader stigmata are cheap
  YARA. Just outside the 24h window (published Kaspersky 07-03) so logged
  here rather than as a fresh item.
  [Securelist](https://securelist.com/tr/armored-likho-apt-with-busysnake-stealer/120292/)
  · [Hacker News writeup](https://thehackernews.com/2026/07/armored-likho-targets-government.html)
  · [TechTimes](https://www.techtimes.com/articles/319680/20260704/new-apt-group-hits-power-grids-three-countries-ai-crafted-malware.htm)

- **ConsentFix — browser-native ClickFix variant against M365 OAuth.**
  Push Security / BleepingComputer covered the technique earlier in the
  week (BleepingComputer story dated ~2026-07-02). Fake Cloudflare Turnstile
  → legitimate Microsoft consent flow → OAuth authorization code exfiltrated
  from the localhost redirect → token + refresh token in ~3 seconds. Bypasses
  MFA (even phishing-resistant MFA) because the attacker steals the code
  post-auth. Abuses pre-consented first-party apps (Azure CLI, Azure
  PowerShell, Visual Studio). Mitigation is Entra Token Protection (PoP
  binding) — enforce it if you can. This is the successor pattern to the
  ARToken / EvilTokens ecosystem covered on 07-04.
  [BleepingComputer](https://www.bleepingcomputer.com/news/security/consentfix-and-clickfix-how-microsoft-365-accounts-are-hijacked-in-3-seconds/)
  · [Push Security](https://pushsecurity.com/blog/consentfix)
  · [CSO Online](https://www.csoonline.com/article/4105230/meet-consentfix-a-new-twist-on-the-clickfix-phishing-attack.html)

- **BioShocking — prompt-injection kill-chain against agentic browsers.**
  Reported earlier in the week (BleepingComputer / Hacker News coverage dated
  ~2026-07-02). Attacker-controlled web page presents a "game" that the
  agent is asked to solve; the game's rules explicitly tell the agent that
  usual safety rules don't apply, and instruct it to fetch and exfiltrate
  credentials from a GitHub repo. Tested successfully against ChatGPT Atlas,
  Comet, Fellou, Genspark, Sigma, and Claude's Chrome plugin; only OpenAI
  shipped a working mitigation. Perplexity closed the report without a fix;
  Anthropic's patch is reported ineffective. Precedent-setter for how
  telco-facing enterprise browsers will be jailbroken. Not in-window today
  but continuing.
  [Malwarebytes](https://www.malwarebytes.com/blog/ai/2026/07/bioshocking-when-gaming-ai-agents-is-no-longer-a-game)
  · [Hacker News](https://thehackernews.com/2026/06/new-bioshocking-attack-tricks-ai.html)

- **MeetingTV v. Palo Alto Networks / Koi Security (filed 2026-06-29,
  motion-to-dismiss response due 2026-07-15).** Startup alleges a Koi
  Security threat report generated with "Wings" + an LLM hallucinated

  · [The Register](https://www.theregister.com/legal/2026/07/02/startup-sues-palo-alto-networks-koi-security-saying-an-ai-hallucinated-report-falsely-linked-it-to-chinese-espionage/5266201)
