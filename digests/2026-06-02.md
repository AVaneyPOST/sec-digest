## General high-severity
- **Miasma — supply-chain compromise of @redhat-cloud-services npm packages (Wiz, 2026-06-01).** 32 packages republished with backdoored versions inside a 72-second window using the project's own GitHub Actions OIDC tokens — i.e. the malicious releases ship with valid SLSA provenance attestations and look legitimate. Each package embeds a `preinstall` hook executing a 4.2 MB obfuscated payload before any application code runs. The payload is the Mini Shai-Hulud worm (TeamPCP), credential-stealing across AWS, GCP, Azure, Kubernetes, HashiCorp Vault, npm and CircleCI; persistence via `kitty-monitor.service` (Linux) and `com.user.kitty-monitor.plist` (macOS); hooks injected into Claude, Codex, Gemini, Copilot, Kiro and opencode tooling. A `gh-token-monitor` process watches the stolen GitHub token and, if the user revokes it before persistence is removed, runs destructive cleanup including wiping `$HOME`. Aggregate ~80k–117k weekly downloads across the affected packages. [Wiz](https://www.wiz.io/blog/miasma-supply-chain-attack-targeting-redhat-npm-packages), [Red Hat RHSB-2026-006](https://access.redhat.com/security/vulnerabilities/RHSB-2026-006), [BleepingComputer](https://www.bleepingcomputer.com/news/security/red-hat-npm-packages-compromised-to-steal-developer-credentials/), [ReversingLabs](https://www.reversinglabs.com/blog/red-hat-cloud-service-npm-packages-backdoored-in-72-seconds), [StepSecurity](https://www.stepsecurity.io/blog/multiple-redhat-cloud-services-npm-packages-compromised), [The Register](https://www.theregister.com/security/2026/06/01/shai-hulud-malware-infects-red-hat-npm-packages-downloaded-80k-times-weekly/5249803), [The Hacker News](https://thehackernews.com/2026/06/miasma-supply-chain-attack-compromises.html) — 2026-06-01 — **CRITICAL**.
- **Windows Netlogon CVE-2026-41089 (CVSS 9.8) — pre-auth RCE on domain controllers, exploitation now mainstream.** Stack-based buffer overflow in the Netlogon RPC parser; an unauthenticated attacker reaches a DC over the network, no user interaction, automated-exploitation-ready. Microsoft patched at May Patch Tuesday (2026-05-12). The Centre for Cybersecurity Belgium updated its May Patch Tuesday advisory on 2026-05-29 to flag active exploitation; on **2026-06-01** Help Net Security, BleepingComputer and SecurityWeek published broad in-the-wild confirmations and 0patch shipped micropatches for unsupported Server versions. Anything Server 2012R2/2016/2019/2022/2025 acting as a DC is in scope; patch DCs first, then audit Netlogon Secure Channel events. [Help Net Security (2026-06-01)](https://www.helpnetsecurity.com/2026/06/01/windows-netlogon-rce-exploited-cve-2026-41089/), [BleepingComputer](https://www.bleepingcomputer.com/news/microsoft/critical-windows-netlogon-remote-code-execution-flaw-now-exploited-in-attacks/), [SecurityWeek](https://www.securityweek.com/critical-windows-netlogon-vulnerability-in-attackers-crosshairs/), [CCB Belgium advisory](https://ccb.belgium.be/advisories/warning-microsoft-patch-tuesday-may-2026-patches-118-vulnerabilities-16-critical-102), [0patch / Malware News](https://malware.news/t/micropatches-released-for-windows-netlogon-remote-code-execution-vulnerability-cve-2026-41089/107332) — 2026-06-01 (escalation/confirmation) — **CRITICAL**.
- **PAN-OS GlobalProtect CVE-2026-0257 — CISA KEV federal deadline passed today.** Authentication-override-cookie bypass on the GlobalProtect portal/gateway; the right cert config plus a forged cookie lands an unauthenticated attacker as the local admin. Rapid7 has been tracking two attack waves (Vultr from 2026-05-18, Dromatics Systems from 2026-05-21). Federal civilian-agency remediation due **2026-06-01** per CISA KEV — i.e. today's news item is the deadline itself. Carry-forward from yesterday's digest; flagged because the operational state has now shifted from "patch this" to "anyone unpatched at COB 2026-06-01 is presumptively-compromised territory." [Palo Alto advisory](https://security.paloaltonetworks.com/CVE-2026-0257), [Rapid7 ETR](https://www.rapid7.com/blog/post/etr-rapid7-observed-exploitation-of-pan-os-globalprotect-authentication-bypass-vulnerability-cve-2026-0257/), [BleepingComputer](https://www.bleepingcomputer.com/news/security/palo-alto-globalprotect-vpn-auth-bypass-flaw-now-exploited-in-attacks/), [The Hacker News](https://thehackernews.com/2026/05/pan-os-globalprotect-authentication.html) — KEV deadline 2026-06-01 — **HIGH**.
- **Operation Dragon Weave — China-aligned APT, Azure Blob Storage C2 (The Hacker News, 2026-06-01 coverage of Seqrite report).** Seqrite published the campaign report on 2026-05-29 (so just outside window for the primary disclosure) but The Hacker News and several aggregators broke it to the wider audience on **2026-06-01**. Spear-phish → ZIP → Rust loader → `AZUREVEIL` AdaptixC2 agent (36 post-exploitation commands) using Microsoft Azure Blob Storage as a dead-drop C2 channel. Targets: government, research, academic, technology and financial services in Czech Republic and Taiwan; earlier iterations seen against Panama (Dec 2025), Cambodia and South Korea (Jan 2026, with Cobalt Strike rather than AdaptixC2). The decoy doc is a ČSSZ (Czech Social Security Administration) appointment notice. Useful as IoC fodder for any operator running Azure-egress monitoring. [Seqrite blog](https://www.seqrite.com/blog/operation-dragon-weave-uncovering-a-china-linked-campaign-targeting-czech-republic-and-taiwan-using-azure-cloud-c2/), [The Hacker News](https://thehackernews.com/2026/06/china-aligned-groups-ramp-up-attacks.html) — 2026-06-01 (wider coverage) — **NOTABLE**.

## Worth watching
- **C2 ISAC operations launch.** Eight US carriers (AT&T, Charter, Comcast, Cox, Lumen, T-Mobile, Verizon, Zayo) announced the Communications Cybersecurity ISAC on 2026-05-26 with operations starting this month. Explicit framing is post–Salt Typhoon cross-carrier sharing on signalling/CALEA/network-edge intrusions. Watch for whether any technical IoCs ever reach the open community. [AT&T release](https://about.att.com/story/2026/c2-isac.html), [Cybersecurity Dive](https://www.cybersecuritydive.com/news/telecom-cybersecurity-c2-isac-launch/820553/), [Industrial Cyber](https://industrialcyber.co/critical-infrastructure/us-telecom-giants-launch-c2-isac-to-strengthen-cyber-intelligence-sharing-protect-communications-infrastructure/).
- **Salt Typhoon — incomplete remediation** continues to be the working assumption for any US carrier threat model. AT&T and Verizon publicly claim eradication; multiple independent analyses and Senate oversight statements through 2026 indicate carriers have not been able to prove eradication. Baseline assumption: long-dwell PRC actor with router-plane persistence.
- **Citizen Lab "Bad Connection" SS7/Diameter surveillance research** (2026-04-23) — still the best public attribution-ish primer on commercial surveillance abuse of interconnect: queries traced to ghost MVNOs piggybacking real telecom networks, combined 3G SS7 + 4G Diameter location-tracking plus malicious SMS with hidden SIM-card commands. Useful baseline for any offensive-telco engagement scoping. [Citizen Lab](https://citizenlab.ca/research/uncovering-global-telecom-exploitation-by-covert-surveillance-actors/).
- **CIFSwitch (Linux kernel CIFS LPE)** — disclosed 2026-05-28 with PoC at `github.com/manizada/CIFSwitch`. Improper validation of `cifs.spnego` key descriptions lets any unprivileged local user impersonate kernel-originated requests and pivot through `cifs.upcall` (root) to local root. Affects Linux Mint, CentOS Stream 9, Rocky Linux 9, Kali and other major distros running `cifs-utils` 6.14+. Code path latent since 2007. Not in window for primary disclosure but the practical risk picture only matured on/around 2026-06-01 with the PoC weaponised across distros; relevant if you have any Linux fleet that mounts CIFS shares (jump hosts, kiosk, dev VMs). [Asim Manizada's writeup](https://heyitsas.im/posts/cifswitch/), [SecurityWeek](https://www.securityweek.com/19-year-old-linux-kernel-vulnerability-exposes-systems-to-root-access/amp/), [BleepingComputer](https://www.bleepingcomputer.com/news/security/new-cifswitch-linux-flaw-gives-root-on-multiple-distributions/), [AlmaLinux test call](https://almalinux.org/blog/2026-05-28-cifswitch/).
- **Cisco Catalyst SD-WAN CVE-2026-20182 (CVSS 10.0)** — `vdaemon` peer-auth bypass on DTLS/UDP 12346 when peer claims `vHub`; cert validation skipped while peer is still marked authenticated. UAT-8616 (Talos) actively dropping SSH keys, NETCONF mods and root escalation. Patched 2026-05 but still being scanned. Carriers running Catalyst SD-WAN management on internet-reachable interfaces are the obvious target set. [Cisco PSIRT advisory](https://sec.cloudapps.cisco.com/security/center/content/CiscoSecurityAdvisory/cisco-sa-sdwan-rpa2-v69WY2SW), [Talos](https://blog.talosintelligence.com/sd-wan-ongoing-exploitation/), [Rapid7](https://www.rapid7.com/blog/post/ve-cve-2026-20182-critical-authentication-bypass-cisco-catalyst-sd-wan-controller-fixed/).
- **NGINX Rift (CVE-2026-42945)** — 18-year-old heap overflow in `ngx_http_rewrite_module`; PoC public, F5 NGINX flagged by VulnCheck since 2026-05-16. RCE in real-world deployments dampened by ASLR, but worker-crash DoS is reliable. Patched 1.31.1 / 1.30.1 / NGINX Plus R36+. Telecom edge proxies and CDN frontends frequently ship vendored NGINX; version-audit. [Help Net Security](https://www.helpnetsecurity.com/2026/05/18/ngnix-vulnerability-exploited-cve-2026-42945/), [SecurityOnline (PoC)](https://securityonline.info/nginx-rce-vulnerability-cve-2026-42945-poc-disclosure/), [Orca Security](https://orca.security/resources/blog/nginx-rewrite-module-vulnerability-cve-2026-42945/).
- **Apache Airflow cluster (CVE-2026-49298, -48726, -46764, -45426, -45360, -42359)** posted to oss-security 2026-05-31. Headline is **CVE-2026-42359 — authenticated RCE via XCom PATCH endpoint**. Vendor severity not yet centrally published. Upgrade path: 3.2.x. Relevant for any multi-tenant Airflow (data platforms in larger telcos). [oss-security 2026/05](https://www.openwall.com/lists/oss-security/2026/05/), [Apache Airflow security](https://airflow.apache.org/docs/apache-airflow/stable/security/index.html).
- **Microsoft Defender CVE-2026-41091 / CVE-2026-45498** — KEV-listed, federal **deadline 2026-06-03** (i.e. tomorrow as of this digest). LPE-to-SYSTEM and Defender-DoS respectively; Defender Platform 1.1.26040.8 / 4.18.26040.7 ships the fix. Carry-forward; the operational point is the deadline. [The Hacker News](https://thehackernews.com/2026/05/microsoft-warns-of-two-actively.html), [Help Net Security](https://www.helpnetsecurity.com/2026/05/21/microsoft-defender-vulnerabilities-cve-2026-41091-cve-2026-45498/).
