# Security digest — 2026-07-15

Window: last 24h ending ~2026-07-15 04:00 UTC.

## TL;DR
- **Microsoft July Patch Tuesday (14 Jul) is the largest in Microsoft's history — ~570 flaws (622 unique CVEs across the catalog), 3 zero-days disclosed, 2 actively exploited before patch: CVE-2026-56164 (SharePoint Server EoP, unauth over network) and CVE-2026-56155 (AD FS EoP, exploited in the wild — discovered by Microsoft DART in incident response).** 59 Critical (48 RCE), including four Windows Media Foundation RCEs (CVE-2026-56189 / -57087 / -57094 / -57090), two SQL Server RCEs (CVE-2026-54118 / -54117), two SharePoint RCEs (CVE-2026-58644 / -50522), and a SharePoint security-feature-bypass (CVE-2026-55040). AD FS fix requires KB5121391 hardening on the Distributed Key Manager container in addition to the rollup.
- **CISA added CVE-2026-56164 to KEV on 14 Jul and issued a same-day SharePoint hardening alert** ("CISA Urges SharePoint Hardening After New Exploitations") calling out active exploitation of three chained SharePoint flaws — CVE-2026-32201, CVE-2026-45659 and the new CVE-2026-56164 — for RCE + IIS-machine-key theft + persistence. Enable AMSI in Full Mode, hunt IIS key theft before rotating.
- **Cisco 15 Jul advisory bundle publishes today.** Advance-notice `cisco-sa-notice-ILh3ZrP5` schedules multiple ISE advisories including unauthenticated root RCE (`cisco-sa-ise-unauth-rce-ZAd2GnJ6`), authenticated RCE (`cisco-sa-ise-rce-4fverepv`), authenticated RCE + path traversal (`cisco-sa-ise-rce-traversal-8bYndVrZ`), and RoomOS DoS + information disclosure (`cisco-sa-tce-roomos-dos-9V9jrC2q`, `cisco-sa-roomos-inf-disc-qGgsbxAm`). No workarounds listed. ISE is deployed across most telco enterprise NAC estates; unauth-root-RCE is the ISE bug pattern that has repeatedly ended up on KEV inside weeks of publication.
- **Adobe patched 7 CVSS 10.0 flaws in ColdFusion and Campaign Classic — CVE-2026-48282 (ColdFusion path traversal) already exploited in the wild within hours of publication** (single attempt from India-geolocated IP, per Adobe/Qualys write-ups). Adobe also formally shifted to a twice-monthly (2nd + 4th Tuesday) release cadence effective this drop, citing AI-accelerated vulnerability discovery.
- **Progress ShareFile — the "credible external threat" of the 10-13 Jul emergency now has a confirmed root cause: authenticated path-traversal zero-day in Storage Zone Controller 5.x + 6.x.** Patches 5.12.5 and 6.0.2 shipped 14 Jul; CVE is reserved but will be published in ~2 weeks per Progress. Progress says no evidence of customer data access. Standard-cloud ShareFile accounts unaffected; hybrid SZC estate should update now, keep servers off if not patched.

## General high-severity

CVE-2026-56164 is unauthenticated network EoP against SharePoint Subscription / 2019 / 2016 — CVSS 5.3 but under active exploitation and now on KEV; AMSI Full Mode blunts. CVE-2026-56155 is authenticated local EoP against AD FS discovered by Microsoft DART in an active incident, CVSS 7.8 — patch alone is insufficient without KB5121391 hardening on the DKM container ACL. 59 Critical total (48 RCE): SharePoint RCE (CVE-2026-58644, -50522), security-feature-bypass (CVE-2026-55040); SQL Server RCE (CVE-2026-54118, -54117); Windows Media Foundation RCE (CVE-2026-56189, -57087, -57094, -57090). DHCP / Exchange / SQL / Hyper-V flagged for second-wave triage. ## Worth watching
- **CrashStealer — new macOS infostealer with valid Apple-notarised signed dropper, Jamf analysis 13-14 Jul.** Native C++ implementation (not the usual AppleScript wrapper), delivered as "Werkbit.app" on a signed disk image that passes Gatekeeper. Targets Chromium browsers, Firefox creds, ~80 crypto-wallet extensions (MetaMask, Phantom, Coinbase, Trust, Rabby, Exodus), 14 password managers (1Password, Bitwarden, LastPass, Dashlane, KeePassXC). AES-GCM-encrypted exfil over libcurl. Apple revoked the associated developer ID after disclosure. First macOS infostealer to combine a valid notarised signature + password managers + wallet extensions at this scale. Sources: [The Hacker News](https://thehackernews.com/2026/07/crashstealer-macos-malware-uses.html), [Jamf Threat Labs](https://www.jamf.com/blog/crashstealer-macos-infostealer-analysis/), [BleepingComputer](https://www.bleepingcomputer.com/news/security/new-crashstealer-malware-poses-as-apple-crash-reporting-tool/), [Help Net Security](https://www.helpnetsecurity.com/2026/07/14/crashstealer-macos-infostealer-password-theft/), [Security Affairs](https://securityaffairs.com/195278/malware/crashstealer-new-macos-infostealer-uses-signed-apps-to-evade-gatekeeper.html).
