## Telecom

- **Cisco Catalyst SD-WAN Manager — CVE-2026-20262, CISA KEV-added 2026-06-15, exploited in the wild [CRITICAL].** Insufficient validation of user-supplied input during file uploads on an API endpoint of Catalyst SD-WAN Manager → low-priv authenticated remote attacker executes arbitrary commands as **root** via crafted HTTP requests. Cisco PSIRT learned of exploitation earlier in June; affects **all** deployment types — on-prem, SD-WAN Cloud-Pro, SD-WAN Cloud (Cisco Managed), and SD-WAN for Government (FedRAMP). Fixes available. IOCs: log entries showing index.jsp or .war file upload attempts in `vmanage-server`, `vmanage-appserver`, and `serviceproxy-access` logs. Practitioner relevance: SD-WAN Manager is the control plane for an operator/MSP customer fleet — root on Manager equals fleet-wide config push. This is now the eighth Cisco SD-WAN bug exploited in 2026 (CVE-2026-20245 from June 5 is still unpatched and discussed separately). Patch this one immediately; carrier and managed-SD-WAN tenants should both treat any internet-reachable SD-WAN Manager as priority-1. [CISA KEV alert — 2026-06-15](https://www.cisa.gov/news-events/alerts/2026/06/15/cisa-adds-two-known-exploited-vulnerabilities-catalog) · [BleepingComputer — Cisco fixes vManage flaw](https://www.bleepingcomputer.com/news/security/cisco-fixes-sd-wan-vmanage-flaw-exploited-in-zero-day-attacks/) · [The Register — make-me-root bug under attack (2026-06-15)](https://www.theregister.com/patches/2026/06/15/cisco-sd-wan-make-me-root-bug-under-attack/5255916) · severity **CRITICAL**.

## General high-severity

- **LiteSpeed cPanel Plugin CVE-2026-54420 — CISA KEV-added 2026-06-15 [HIGH].** UNIX Symbolic Link (Symlink) Following (CWE-61), CVSS **8.5**. LiteSpeed cPanel plugin before 2.4.8 (distributed in LiteSpeed WHM PlugIn before 5.3.2.0) mishandles symlinks provided by users with FTP or web-shell access on shared hosting servers running CloudLinux/CageFS — exploited in the wild as early as May 2026, public disclosure June 14. Impact is tenant-isolation bypass: users with FTP or web-shell access on a shared hosting server can use symlinks to read other customers' data outside their document root. Federal due date under BOD 26-04: 2026-07-06. Fix: upgrade WHM Plugin to 5.3.2.0 or later. Relevance to a telco offensive-security practitioner: limited unless you run shared hosting; broader interest is the steady tempo of CISA mandates against plugin-class bugs that telcos historically deprioritise. [CISA KEV alert — 2026-06-15](https://www.cisa.gov/news-events/alerts/2026/06/15/cisa-adds-two-known-exploited-vulnerabilities-catalog) · [SecurityWeek — CISA urges immediate patching](https://www.securityweek.com/cisa-urges-immediate-patching-of-exploited-litespeed-cpanel-plugin-zero-day/) · [Halo Security analysis](https://blog.halosecurity.com/cve-2026-48172-litespeed-cpanel-plugin-privilege-escalation/) · severity **HIGH**.

- **Microsoft 365 Copilot Enterprise Search — "SearchLeak" CVE-2026-42824, Varonis disclosure 2026-06-15 [HIGH, backend-patched].** Three-bug chain: (1) parameter-to-prompt (P2P) injection via the URL `q` parameter that feeds Copilot attacker-supplied instructions; (2) HTML render race condition fires an attacker-controlled `<img>` tag before output sanitisation completes; (3) Bing server-side request forgery routes the stolen data through Bing's image retrieval endpoint, bypassing the page's CSP because the request originates from Microsoft infrastructure. Exfiltrates mailbox, calendar, OneDrive, SharePoint contents — and MFA codes from email. The victim observes only "Copilot thinking…". Microsoft rated **critical**, CVSS scores 6.5 (MSRC) / 7.5 (NVD). Backend-patched in early June 2026; no customer action required. Varonis reports PoC only, no in-the-wild exploitation observed. Practitioner takeaway: trust-boundary mistakes in agentic-LLM products are still being shipped to enterprise GA; the architectural lesson — "Microsoft-origin requests bypass tenant CSP" — generalises far beyond this one product. [Varonis — SearchLeak write-up](https://www.varonis.com/blog/searchleak) · [The Hacker News (2026-06-15)](https://thehackernews.com/2026/06/one-click-microsoft-365-copilot-flaw.html) · [Dark Reading — Copilot 'SearchLeak' 1-click data theft](https://www.darkreading.com/application-security/copilot-searchleak-attack-1-click-data-theft) · [BleepingComputer](https://www.bleepingcomputer.com/news/security/new-attack-turned-microsoft-365-copilot-into-1-click-data-theft-tool/) · severity **HIGH** (architectural; product itself already remediated).

- **Oracle CPU — June 2026 (released 2026-06-16), 251 patches [HIGH].** Headline general-enterprise items beyond the Communications products called out above: Oracle Enterprise Manager top severity CVSS **9.9** (6 of 16 EM bugs remotely exploitable without authentication); 106 patches across Oracle Fusion Middleware; ongoing PeopleSoft attention given last week's CVE-2026-35273 KEV-add and ShinyHunters / UNC6240 mass-exploitation. ## Worth watching

- **OptinMonster / TrustPulse / PushEngage CDN supply-chain backdoor — second-wave reporting through June 15.** First malicious activity 2026-06-12; Sansec disclosed 2026-06-13; OptinMonster owner Awesome Motive published its incident note 2026-06-14; PushEngage CDN nodes kept serving the malicious payload through June 14. ~1.2M sites exposed across the three plugins. The payload runs in the browser of any logged-in admin who loads an affected page, creates a hidden admin account, and installs a self-hiding backdoor plugin; new credentials are POSTed to `tidio.cc` (lookalike of tidio.com). Root cause per Awesome Motive: compromised CDN API key. Practitioner takeaway: any operator-adjacent WP estate (MVNO portals, marketing, corp comms) that ran these plugins between June 12–14 needs (a) hidden admin-account audit, (b) plugin-list audit for the self-hiding backdoor, (c) credential and session rotation, (d) outbound block on `tidio.cc`. [Patchstack analysis](https://patchstack.com/articles/supply-chain-attack-on-optinmonster-trustpulse-and-pushengage-tampered-cdn-scripts-auto-creating-rogue-admins/) · [Sansec research](https://sansec.io/research/optinmonster-supply-chain-attack) · [SecurityAffairs — June 14 coverage](https://securityaffairs.com/193616/malware/supply-chain-attack-hits-popular-wordpress-plugins-through-awesome-motive-cdn.html) · [Infosecurity Magazine](https://www.infosecurity-magazine.com/news/wordpress-plugin-supply-chain/).
