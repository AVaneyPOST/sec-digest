# Security digest — 2026-07-18

Window: last 24h ending ~2026-07-18 04:06 UTC.

## TL;DR
- **CISA adds two Fortinet FortiSandbox unauth OS-command-injection flaws (CVE-2026-39808, CVE-2026-25089, both CVSS 9.1) to KEV on 17 Jul with BOD 26-04 deadline Sunday 19 Jul.** Fortinet patched 39808 in April and 25089 in June; Defused Cyber reported live exploitation over the past week alongside a third bug (CVE-2026-39813). FortiSandbox sits in the verdict path of the wider Fortinet Security Fabric, so compromise can invert threat-block decisions across FortiGate/FortiClient/FortiEDR at the tenant. Directly relevant to any MSSP and to telco NOC/SOC estates.
- **WordPress ships 6.9.5 and 7.0.2 (17 Jul) closing "wp2shell", a pre-auth RCE in core (CVE-2026-63030 REST batch-route confusion chained with CVE-2026-60137 SQLi in `author__not_in`).** An anonymous HTTP request pops a default install with no plugins required. Assetnote/Searchlight (Adam Kues) reported it via HackerOne and are holding technical details; a self-check is at wp2shell.com. No mass exploitation yet, but the moment technical details drop the exposed WordPress population is very large.
- **OpenSSL silently ships fixes for "HollowByte" DoS (17 Jul reporting).** An 11-byte TLS handshake header (declared length only, no payload) causes vulnerable OpenSSL to pre-allocate the claimed buffer; combined with glibc's tendency to retain freed medium chunks, waves of randomized-size connects strand memory permanently. Okta's NGINX tests: low-end servers exhaustible, high-end lose ~25% RSS while staying below alerting thresholds. Fixed in 4.0.1 and backported to 3.6.3, 3.5.7, 3.4.6, 3.0.21 — but shipped with no CVE, no advisory, no changelog note.
- **Elastic Security Labs discloses a live North Korea "Contagious Interview" cluster (REF9403) hiding malware in SVG steganography (17 Jul).** Threat actor targeted Elastic's own community Slack `#jobs` channel with a fake e-commerce project; payloads split into base64 fragments inside HTML comments in flag-image SVGs (AE.svg, AF.svg…) load an OTTERCOOKIE-aligned four-stage payload (browser/wallet stealer, file stealer, Socket.IO RAT, clipboard stealer). New initial-access avenue for a campaign that had been trending toward npm/GitHub lures.
- **Oracle E-Business Suite CVE-2026-46817 BOD 26-04 deadline is today, Saturday 18 Jul.** Unauth account takeover in Oracle Payments (12.2.3-12.2.15). 3-day compressed KEV window (added 15 Jul). Actively exploited in the wild since 29 Jun per Defused honeypot telemetry; >1,000 EBS instances internet-exposed. Federal agencies remediating today; enterprise EBS estates without the May 2026 CPU should treat as active.

- **WordPress 6.9.5 / 7.0.2 patch "wp2shell" pre-auth RCE (CVE-2026-63030 + CVE-2026-60137, 17 Jul)** — [CRITICAL] REST API batch-request route confusion chained with SQL injection in the `author__not_in` WP_Query parameter → anonymous unauthenticated RCE on a default install with no plugins. Discovered by Adam Kues at Assetnote / Searchlight Cyber via WordPress's HackerOne program. Details are being withheld; wp2shell.com self-checker published. No advisory carries a CVE or CVSS score officially yet (CVE record not yet appeared on NVD as of writing per Nebula Design coverage), but researcher references CVE-2026-63030 / CVE-2026-60137. No exploitation reported through publication. Given the class (SQLi + REST batch confusion, both easy-to-derive from a diff), assume public PoC in days, not weeks. Sources: [The Hacker News](https://thehackernews.com/2026/07/new-wp2shell-wordpress-core-flaw-lets.html), [Searchlight Cyber / Assetnote research](https://slcyber.io/research-center/wp2shell-pre-authentication-rce-in-wordpress-cores-rest-batch-api), [Hadrian (technical writeup)](https://hadrian.io/blog/wp2shell-a-pre-authentication-rce-in-wordpress-cores-rest-batch-api), [Nebula Design](https://nebula.design/security/wp2shell-wordpress-rce/), [Beazley Security advisory BSL-A1193](https://labs.beazley.security/advisories/BSL-A1193), [Cyber Kendra](https://www.cyberkendra.com/2026/07/wp2shell-critical-wordpress-flaw-lets.html).
- **OpenSSL "HollowByte" 11-byte DoS — silent patch across 4.0.1 / 3.6.3 / 3.5.7 / 3.4.6 / 3.0.21 (17 Jul reporting)** — [HIGH] Root cause: OpenSSL pre-allocates the buffer declared in the 4-byte TLS message header before reading the payload. Combined with glibc `malloc` not returning small-to-medium freed chunks to the kernel, a burst of connects with randomised claimed sizes strands memory that the allocator can no longer reuse. Okta demonstrated ~25% RSS erosion on a stock NGINX + OpenSSL server at attack rates well below security telemetry thresholds. No CVE, no security advisory, no changelog entry — patched only in the source tree. Practitioner take: patch OpenSSL 3.x branches immediately (even without a CVE track), instrument RSS trending independently of RPS/HTTP rate. Sources: [BleepingComputer](https://www.bleepingcomputer.com/news/security/hollowbyte-ddos-flaw-bloats-openssl-server-memory-with-11-byte-payload/), [The Hacker News](https://thehackernews.com/2026/07/openssl-hollowbyte-flaw-could-freeze.html), [Okta primary research](https://sec.okta.com/articles/2026/06/openssl-hollowbtye-a-dos-hiding-in-11-bytes/), [Cybersecurity News](https://cybersecuritynews.com/openssl-hollowbyte-vulnerability/).
- **Elastic Security Labs — REF9403 Contagious Interview SVG steganography campaign (17 Jul)** — [HIGH] North Korean cluster (aligned with the Contagious Interview / OTTERCOOKIE lineage) targeted Elastic's community Slack workspace `#jobs` channel with a fake e-commerce integration project. The trap: a repo whose `assets/` directory contains country-flag SVGs (AE.svg, AF.svg, …), each carrying base64-encoded malware fragments inside HTML comment blocks; the loader reassembles across SVGs. Four-stage payload: browser credential + crypto wallet stealer → file stealer → Socket.IO RAT → clipboard stealer. Notable initial-access shift for Contagious Interview — previously npm packages, coding-challenge repos, LinkedIn recruiter lures; the community-Slack angle is broadly reusable against any developer-facing community with an unmoderated jobs channel. Sources: [Elastic Security Labs primary](https://www.elastic.co/security-labs/contagious-interview-malware-svg-steganography), [The Hacker News](https://thehackernews.com/2026/07/north-korea-linked-hackers-hide.html), [Infosecurity Magazine](https://www.infosecurity-magazine.com/news/north-korean-contagious-interview/).
