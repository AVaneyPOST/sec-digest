## General high-severity

- **PAN-OS GlobalProtect CVE-2026-0257 — auth-bypass cookie forgery, added to CISA KEV** — CISA added the CVE to KEV on **2026-05-29**. Palo Alto disclosed on 2026-05-13 and updated the advisory on 2026-05-29 alongside the KEV addition. **Bug:** when a GlobalProtect portal/gateway reuses the same certificate to encrypt/decrypt auth-override cookies as another feature exposes that key publicly, an attacker can recover the public key, **forge a valid authentication-override cookie**, and use it to establish a VPN session as an arbitrary user — no creds needed. **Exploitation in the wild:** Rapid7 MDR first observed exploitation on **2026-05-17**, second wave **2026-05-21**, same MAC across both → believed same actor; in the second wave the attacker received a VPN IP assignment after cookie auth, i.e. **actually got onto internal networks** (no lateral movement observed yet by Rapid7). NVD scored this medium; Rapid7 urges treating it as critical given the pre-auth network-edge nature. Fix in PAN-OS updates; mitigations: disable the auth-override feature or generate a new, exclusive certificate for it. [Rapid7 writeup (2026-05-29)](https://www.rapid7.com/blog/post/etr-rapid7-observed-exploitation-of-pan-os-globalprotect-authentication-bypass-vulnerability-cve-2026-0257/) · [Palo Alto advisory](https://security.paloaltonetworks.com/CVE-2026-0257) · [NVD entry](https://nvd.nist.gov/vuln/detail/CVE-2026-0257) · [SentinelOne summary](https://www.sentinelone.com/vulnerability-database/cve-2026-0257/) · Date: 2026-05-29 (KEV add + Rapid7 publication) · **CRITICAL**

- **Marimo CVE-2026-39987 — first observed LLM-agent post-exploitation chain** — Sysdig Threat Research published the campaign writeup on **2026-05-29**. The CVE itself (pre-auth RCE in Marimo ≤0.20.4 via unauthenticated `/terminal/ws` WebSocket → full PTY) was disclosed April 2026 and is exploited within ~10h of disclosure; the new bit is what came after. On **2026-05-10**, an intruder ran an LLM agent in the post-exploit phase that: (1) found and extracted two cloud credentials on the compromised host, (2) replayed them via a fan-out pool of Cloudflare Workers, (3) pulled an SSH private key out of AWS Secrets Manager, (4) launched 8 short, parallel SSH sessions against an internal bastion, and (5) exfiltrated full schema + contents of an internal Postgres DB **in under 2 minutes**. Direction-of-travel piece: the first publicly documented agentic AI driving real lateral movement and exfil, not just lure generation or recon. Patch Marimo, audit for internet-facing instances, rotate creds/SSH keys. [Sysdig research](https://webflow.sysdig.com/blog/ai-agent-at-the-wheel-how-an-attacker-used-llms-to-move-from-a-cve-to-an-internal-database-in-4-pivots) · [The Hacker News](https://thehackernews.com/2026/05/attackers-use-llm-agent-for-post.html) · [Cybersecurity News](https://cybersecuritynews.com/hackers-use-llm-agent-to-move-from-marimo-rce/) · Date: 2026-05-29 · **HIGH**

- **Dokploy CVE-2026-45663 — authenticated container-file-upload → host RCE (CVSS 9.9)** — Published 2026-05-29. In Dokploy ≤0.29.1, the Docker file-upload feature interpolates a user-supplied `destinationPath` directly into a shell command for `docker cp`. Shell metacharacters (`;`, `"`, etc.) escape the intended command and execute as arbitrary OS commands on the **Dokploy host**, not just inside a container — i.e. straight container-to-host break. Auth required but trivial in any multi-tenant or shared-team install. No public PoC yet. Upgrade. [TheHackerWire advisory](https://www.thehackerwire.com/dokploy-critical-command-injection-cve-2026-45663/) · [Vulnerability-Lookup entry](https://vulnerability.circl.lu/vuln/cve-2026-45663) · Date: 2026-05-29 · **HIGH**

- **Malicious `Sicoob.Sdk` NuGet supply-chain attack — Brazilian banking creds exfiltrated** — Socket's writeup amplified across the wire on **2026-05-29**. Package masquerades as the C# SDK for **Sicoob** (one of Brazil's largest cooperative banking groups). Versions **2.0.0–2.0.4** published 2026-05-05/06 and downloaded ~500 times before NuGet pulled them. Trigger path: when a dev instantiates `SicoobClient(clientId, pfxPath, pfxPassword)` the SDK reads the PFX file off disk, base64-encodes it, and POSTs `{clientId, pfxPassword, pfxData}` to a hard-coded third-party **Sentry** endpoint. PFX certs are the auth primitive for the Sicoob banking network → instant pivot to bank-side fraud. A **parallel npm campaign** disclosed the same day targets AWS / CI-CD secrets. Anyone who installed any 2.0.x of `Sicoob.Sdk`: treat the PFX material as fully compromised, revoke certs, rotate PFX passwords, and disable the associated client IDs at Sicoob. [The Hacker News](https://thehackernews.com/2026/05/malicious-sicoob-nuget-steals-banking.html) · [Socket research](https://socket.dev/blog/malicious-nuget-package-impersonates-sicoob-sdk) · [Cybersecurity News](https://cybersecuritynews.com/malicious-nuget-sicoob-sdk/) · Date: 2026-05-29 · **HIGH**
