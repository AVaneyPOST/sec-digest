## Worth watching

- **Check Point IKEv1 VPN authn bypass (CVE-2026-50751, CVSS 9.3) — Qilin ransomware actively exploiting.** Affects deployments still on the deprecated IKEv1 key exchange (Mobile Access / SSL VPN / Remote Access / Spark FW); a logic flaw in cert validation lets an unauthenticated attacker establish a VPN session without a valid password. Exploitation since 2026-05-07, surged early June, ~"a few dozen" orgs hit, at least one tied to Qilin. CISA added to KEV 2026-06-08 with a 2026-06-11 federal deadline (just outside our strict 24h window — listing here so it doesn't slip). Hotfix shipped; the durable fix is killing IKEv1.
  - https://www.bleepingcomputer.com/news/security/check-point-links-vpn-zero-day-attacks-to-qilin-ransomware-gang/
  - https://www.securityweek.com/check-point-vpn-zero-day-exploited-in-qilin-ransomware-attacks/
  - KEV-add date: 2026-06-08. [CRITICAL]

- **LiteLLM CVE-2026-42271 — KEV add 2026-06-08.** First AI-infrastructure CVE to hit KEV this cycle; federal mitigation clock now running. Relevant if anyone in your org is brokering model calls through LiteLLM as an internal API gateway. [HIGH]

- **Chrome V8 zero-day CVE-2026-11645 — emergency patch 2026-06-08.** Out-of-bounds R/W in V8 → arbitrary code in renderer sandbox via crafted page. Fifth Chrome 0day of 2026. Stable: 149.0.7827.102/.103. Patched just before the window opened; flagged because rollout is gradual and exposure remains. [HIGH]
  - https://thehackernews.com/2026/06/chrome-v8-zero-day-cve-2026-11645.html
  - https://www.helpnetsecurity.com/2026/06/09/google-chrome-zero-day-cve-2026-11645/

- **Verizon VoLTE / IMS — CERT/CC VU#615987 / CVE-2026-10629 (disclosed 2026-06-02, ~8 days old).** Primary telecom item I want to keep flagged: SIP signaling on Verizon's IMS observed transmitting without IPsec ESP encapsulation and without SIP Security Agreement headers (no Security-Client/Server/Verify in REGISTER). On-path attackers can intercept, replay, or modify VoLTE signalling — call hijacking, spoof, DoS, manipulation of emergency-call routing all in scope. iOS 26.5 carrier bundle (released 2026-05-11) added IMS-IPsec settings, but presence ≠ active enforcement. Verizon's response: "Not Affected" — argues 3GPP/GSMA provisions aren't mandatory. Worth tracking for any follow-up from GSMA or other US carriers. Offensive-security angle: classic on-path SIP manipulation primitives are back in scope on a tier-1 US operator. [NOTABLE → HIGH if confirmed beyond test conditions.]
  - https://kb.cert.org/vuls/id/615987 (CERT/CC kb — WebFetch 403, refer to mirror/coverage)
  - https://cyberinsider.com/verizon-volte-network-found-missing-ipsec-protections-for-sip-signaling/

- **Cisco SD-WAN Manager CVE-2026-20245 (disclosed 2026-06-05) — root EoP via authenticated crafted file upload + command injection.** Cisco PSIRT accelerated disclosure on evidence of ITW abuse; exploit chain appears tied to prior SD-WAN-compromise activity. Auth required but post-foothold this is straight to root on the Manager. Relevant for telco/enterprise SD-WAN estates. [HIGH]

- **Salt Typhoon — political pressure cycle continues.** Sen. Cantwell publicly stated AT&T/Verizon are blocking release of Mandiant network-security assessments tied to remediation; carriers haven't produced documentation that the intrusion has been eradicated. No new technical disclosure, but the policy/CVD-pipeline pressure is shaping what regulators may demand next. Worth tracking even though there's no fresh in-window event.
  - https://www.nextgov.com/cybersecurity/2026/02/senator-says-t-and-verizon-blocked-release-salt-typhoon-security-reports/411172/

- **HTTP/2 Bomb (oss-security, 2026-06-03).** Remote DoS affecting Apache httpd, nginx, IIS, Envoy, Cloudflare Pingora in default HTTP/2 configs. Outside strict window; mentioned because it touches every operator-facing reverse proxy / load balancer in the fleet and patches are still rolling.
