# Security digest — 2026-06-13

Window: last 24h, ending 2026-06-13 04:04 UTC. A telecom-heavy day, anchored by Sygnia's
disclosure that a China-nexus group sat inside a telco's Linux login stack for nearly a
decade, and by Google's lawsuit naming a Gemini-powered smishing-as-a-service operation.

## TL;DR

- **Velvet Ant ("Operation Highland")** — Sygnia disclosed a ~10-year telco intrusion in
  which the actor backdoored `pam_unix.so` and OpenSSH instead of dropping malware.
  Earliest artifacts trace to 2016. Direct relevance to anyone testing telco internal
  AAA/management planes.
- **Google v. "Outsider Enterprise"** — Google filed (NY SDNY) against a China-based
  phishing-kit-as-a-service ring that used Gemini to mass-generate landing pages and
  flood ~2.5M SMS to Android users; emergency injunction granted. Sets a precedent on
  LLM-abuse in SMS interconnect fraud.
- **LangGraph RCE chain** — Three CVEs in self-hosted langgraph deployments
  (CVE-2025-67644 SQLi + CVE-2026-28277 unsafe msgpack deserialization, chainable to
  unauth RCE on SQLite/Redis checkpointer backends).
- **Atomic Arch supply-chain** — 400+ orphaned AUR packages taken over and re-armed with
  a Rust infostealer that can side-load an eBPF rootkit when built as root.
- **Ivanti Sentry CVE-2026-10520 (CVSS 10.0)** — Unauth pre-auth OS command injection;
  CISA BOD 26-04 mandate (3-day patch deadline) is in force through June 14; PoC public.

## Telecom

- **Sygnia "Operation Highland" — Velvet Ant backdoored Linux PAM and OpenSSH inside a
  telecom network, undetected since 2016.** Instead of new implants, the actor replaced
  `pam_unix.so` and `sshd`/`ssh` with trojanized copies — one set granted login via a
  hardcoded secret, the other logged real credentials and every command typed, with a
  toggle to suppress that logging on demand. The targeted segment had no direct
  internet access; the actor pivoted in via internet-facing systems. For an offensive
  telecom practitioner this is a textbook persistence pattern for the OAM/EMS planes
  that aggregate jumphost access to SS7/Diameter STPs, packet core NFs, billing, and
  HSS/UDM — verify integrity of PAM and OpenSSH binaries on your bastions, not just
  their config. [[Sygnia](https://www.sygnia.co/blog/operation-highland-velvet-ant/)]
  [[The Hacker News](https://thehackernews.com/2026/06/china-linked-hackers-backdoored-linux.html)]
  [[Dark Reading](https://www.darkreading.com/threat-intelligence/chinese-apts-linux-backdoor-telco-attacks)]
  — 2026-06-12 — **CRITICAL**

- **Google sues "Outsider Enterprise" — Gemini-powered smishing-as-a-service that
  pushed 2.5M SMS to Android users.** Complaint filed in SDNY on 2026-06-12;
  emergency injunction granted by Judge Marrero. Google attributes 9,000 fake sites
  and ~1.59M phishing URLs to the kit between 2025-11-14 and 2026-04-14; the operator
  rented the kit for as little as $88/week. Notable for telco fraud teams: this is
  the first time Google has gone after a vendor for misusing the LLM itself as the
  text-generation engine for smishing campaigns, and it ties bulk SMS abuse to a
  named provider rather than a generic "spam" attribution.
  [[The Hacker News](https://thehackernews.com/2026/06/google-sues-chinese-smishing-network.html)]
  [[Help Net Security](https://www.helpnetsecurity.com/2026/06/12/google-china-based-cybercrime-network-lawsuit/)]
  — 2026-06-12 — **HIGH**

## General high-severity

- **LangGraph RCE chain (CVE-2025-67644 + CVE-2026-28277 + CVE-2026-27022).** Yarden
  Porat (research disclosed 2026-06-12) found a SQLi in langgraph (CVSS 7.3), an
  unsafe msgpack deserialization in langgraph-checkpoint (CVSS 6.8), and a RediSearch
  query injection (CVSS 6.5). The first two chain to remote code execution on
  self-hosted deployments using the SQLite or Redis checkpointer with user-controlled
  filter input — i.e. most production agent stacks. Fixed in
  langgraph-checkpoint-sqlite ≥3.0.1, langgraph ≥1.0.10, langgraph-checkpoint-redis
  ≥1.0.2.
  [[The Hacker News](https://thehackernews.com/2026/06/langgraph-flaw-chain-exposes-self.html)]
  — 2026-06-12 — **HIGH**

- **Atomic Arch — 400+ AUR packages hijacked to ship a Rust infostealer with eBPF
  rootkit capability.** Attackers adopted orphaned AUR packages through the standard
  ownership-transfer flow, rewrote PKGBUILDs to pull two rogue npm packages
  (`atomic-lockfile`, `js-digest`), and dropped a Rust stealer targeting Slack, Teams,
  Discord, GitHub/npm tokens, HashiCorp Vault tokens, Docker/Podman credentials,
  SSH/VPN material, shell histories, and browser/Electron data. If the build runs as
  root, an eBPF rootkit is loaded for stealth. Anyone who installed/updated an AUR
  package on or after 2026-06-11 should treat the host as compromised pending list
  reconciliation.
  [[BleepingComputer](https://www.bleepingcomputer.com/news/security/over-400-arch-linux-packages-compromised-to-push-rootkit-infostealer/)]
  [[The Hacker News](https://thehackernews.com/2026/06/over-400-arch-linux-aur-packages.html)]
  — 2026-06-11/12 — **HIGH**

- **CISA BOD 26-04 — 3-day federal patch mandate for Ivanti Sentry CVE-2026-10520
  (CVSS 10.0).** Pre-auth OS command injection in
  `/mics/api/v2/sentry/mics-config/handleMessage`; vendor patches in Sentry R10.5.2 /
  R10.6.2 / R10.7.1. Public PoC dropped 2026-06-10; CISA KEV addition 2026-06-11;
  Shadowserver telemetry shows most internet-exposed Sentries already backdoored.
  Sentry sits on the policy/control path for managed mobile devices, so root on Sentry
  ≈ control of corporate-managed handsets — relevant to anyone modeling MDM as a
  trust anchor for telecom-issued lines.
  [[CISA KEV](https://www.cisa.gov/known-exploited-vulnerabilities-catalog)]
  [[BleepingComputer](https://www.bleepingcomputer.com/news/security/cisa-gives-feds-3-days-to-patch-ivanti-flaw-exploited-in-attacks/)]
  [[Help Net Security](https://www.helpnetsecurity.com/2026/06/10/ivanti-sentry-cve-2026-10520-cve-2026-10523/)]
