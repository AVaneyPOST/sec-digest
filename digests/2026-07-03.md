## Telecom
- **Cisco Unified Communications Manager CVE-2026-20230 — vendor confirms exploitation** — [Bleeping Computer](https://www.bleepingcomputer.com/news/security/cisco-finally-confirms-attackers-exploiting-unified-cm-flaw/) / [SecurityWeek](https://www.securityweek.com/cisco-confirms-in-the-wild-exploitation-of-unified-cm-vulnerability/) / [Cisco advisory](https://www.cisco.com/c/en/us/support/docs/csa/cisco-sa-cucm-ssrf-cXPnHcW.html) — 2026-07-02 — **CRITICAL**. Unauthenticated SSRF against the WebDialer service on Unified CM lets an attacker write arbitrary files to the underlying OS via a crafted HTTP request, deploy a JSP webshell, and land root on the VoIP call-control platform. Exploitation seen since ~June 22 (per Defused); Cisco previously refused to confirm and today did. Shadowserver counts **200+ exposed instances** on the internet. Only vulnerable if WebDialer is enabled (disabled by default, but commonly turned on for click-to-call in enterprise/carrier voice estates). CISA KEV since June 25; BOD 26-04 deadline already passed (June 28). Patch or disable WebDialer. Practitioner note: Unified CM sits deep inside telco/enterprise voice — SIP trunks, call recording, MWI, presence — a compromise here is a foothold into voice fraud, wiretap-style call interception, and CDR access.
- **Nothing new** on the SS7 / Diameter / GTP / SIGTRAN / 5G SBA / N32 / IMS / XCAP / eSIM sides in the 24h window. No GSMA CVD publications, no Ericsson / Nokia / Mavenir / Oracle Comms PSIRT advisories dated 2026-07-02 surfaced.

## General high-severity
- **Microsoft SharePoint Server CVE-2026-45659 — CISA KEV, exploitation observed** — [Bleeping Computer](https://www.bleepingcomputer.com/news/security/cisa-microsoft-sharepoint-rce-flaw-now-actively-exploited/) / [The Hacker News](https://thehackernews.com/2026/07/sharepoint-rce-cve-2026-45659-added-to.html) / [SecurityWeek](https://www.securityweek.com/cisa-warns-of-actively-exploited-microsoft-sharepoint-vulnerability/) — CISA added 2026-07-01, widely reported 2026-07-02 — **HIGH**. Deserialization of untrusted data → RCE for an authenticated user with Site Member (low) permissions. CVSS 8.8. Microsoft patched out-of-band 2026-05-21 across Subscription Edition / 2019 / Enterprise 2016. Federal deadline **2026-07-04** — 48 hours out at time of writing. Microsoft initially labelled exploitation "less likely"; that assessment did not survive contact with the internet.
- **Citrix NetScaler ADC/Gateway CVE-2026-8451 ("CitrixBleed 2") — active exploitation** — [watchTowr Labs writeup](https://labs.watchtowr.com/citrixbleed-to-infinity-and-beyond-citrix-netscaler-pre-auth-memory-overread-cve-2026-8451/) / [The Hacker News](https://thehackernews.com/2026/07/citrix-patches-six-netscaler-flaws.html) / [CyberScoop](https://cyberscoop.com/citrix-netscaler-flaw-cve-2026-8451-citrixbleed/) / [SocRadar](https://socradar.io/blog/cve-2026-8451-netscaler-overread-citrixbleed/) — patched 2026-06-30, exploitation confirmed 2026-07-01/02 — **CRITICAL**. Pre-auth SAML AuthnRequest at `/saml/login` — NetScaler's XML parser stops reading unquoted attributes at NUL/`>` instead of whitespace, allowing memory readback in the auth-cookie response. Payload: malformed `samlp:AuthnRequest` padded with 476 spaces. watchTowr's PoC leaked what look like valid process-memory pointers — pair with a corruption bug and it becomes full compromise. Exposure requires the appliance to be configured as a **SAML Identity Provider**. Lupovis honeypots saw coordinated scan-and-exploit across three sensor clusters within ~5 hours of the first probe. Patch to the June 30 build; if you can't, disable SAML IdP role.
- **Medtronic — data breach notifications begin** — [The Register](https://www.theregister.com/security/2026/07/02/pacemaker-manufacturer-medtronic-warns-patients-cybercrooks-may-have-swiped-health-data/5265768) / [GovInfoSecurity](https://www.govinfosecurity.com/medtronic-notifying-patients-affected-by-data-theft-hack-a-32115) / [HIPAA Journal](https://www.hipaajournal.com/medical-device-maker-medtronic-data-breach/) — 2026-07-02 — **HIGH**. Detected 2026-04-15; corporate IT accessed 2026-04-13 → 04-19. Exfiltrated categories: names, contact details, DOBs, SSNs, health information. ShinyHunters previously claimed 9M records; Medtronic's state-level filings so far: ~64k in MA, ~300k in TX, national total not yet disclosed. Medtronic says no impact to device security / patient safety / manufacturing. No attribution in notification letters despite the ShinyHunters claim.
- **CISA ICS advisories — three published** — [ICSA-26-183-03 (Gardyn IoT Hub)](https://www.cisa.gov/news-events/ics-advisories/icsa-26-183-03) — 2026-07-02 — **NOTABLE**. ## Worth watching
- **Cisco CUCM exposure follow-up** — with 200+ CUCM boxes publicly exposed and PoC-plus-vendor-confirmation now aligned, expect a second exploitation wave targeting non-federal enterprises and MSSP-managed telephony estates over the coming week. Also worth checking whether Cisco Emergency Responder / CUCM IM&P are reachable off the same edge — WebDialer is often bundled behind the same reverse proxy.
- **Chinese-language smishing PhaaS on RCS/iMessage** — [Unit 42 "Smishing Deluge"](https://unit42.paloaltonetworks.com/global-smishing-campaign/) — pre-window but a durable trend cluster; multiple threat-actor teams now competing to deliver via RCS/IM because SMS envelope-based fraud detection doesn't see them. Verified-sender abuse in RCS is the next battleground.
- **CitrixBleed pattern is endemic** — watchTowr flags CVE-2026-8451 as the *third* memory-overread they've stumbled onto in NetScaler while investigating something else (after CVE-2026-3055 and the original CitrixBleed lineage). If you own NetScaler at telecom edge (partner portals, self-service, BSS front-ends) treat memory-overread findings as a class problem, not a one-off.
- **KDDI email breach (Japanese telco)** — [breach retrospectives referring to June 23 disclosure] — 14.22M email addresses/passwords exposed; still worth tracking whether upstream third-party component is used by other telcos.
- **Kigen eUICC / GSMA TS.48 eSIM cloning** — 2025 disclosure but relevant this week as GSMA TS.48 v7.0 mitigations continue rolling out; watch for follow-on research into other eUICC vendors using Java Card.
