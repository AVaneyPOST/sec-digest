Socket caught the first release six minutes after publish. Pin to 8.15.0 or roll back to 8.13.0; clear all affected versions from lockfiles and caches; treat any secrets on affected dev machines as burned.

## General high-severity
- **`jscrambler` official npm package compromised — five malicious releases with Rust infostealer** — [HIGH] Publisher account `jscrambler_` pushed 8.14.0, 8.16.0, 8.17.0, 8.18.0 and 8.20.0 on 11 Jul over ~3 hours, interleaved with clean releases the maintainers appear to have pushed as remediation. A `preinstall` hook drops and executes a native Rust binary — Windows / macOS / Linux builds included — that steals Chrome / Brave / Edge / Chromium profile data, the Bitwarden browser extension state and Steam sessions, and persists via Task Scheduler and LaunchAgents. Exfil to a drop server over TLS. Socket flagged the first version six minutes after publish; StepSecurity confirmed independently. Immediate action: pin to 8.15.0 (post-incident clean release) or 8.13.0 (pre-incident), scrub every affected version from lockfiles and offline caches, rotate any dev-workstation secrets that touched an install since 11 Jul. Sources: [The Hacker News](https://thehackernews.com/2026/07/compromised-jscrambler-8140-npm-release.html), [StepSecurity](https://www.stepsecurity.io/blog/jscrambler-npm-package-publishes-malicious-preinstall-binary), [SafeDep](https://safedep.io/jscrambler-npm-supply-chain-compromise/).
- **"One Target, Two Flags" — China-nexus and India-nexus operators separately compromise Pakistani law enforcement** — [HIGH / espionage] SentinelLABS report published 09 Jul, cross-market coverage (The Hacker News, The Record, TechJuice) landed 10-11 Jul in-window. Four distinct clusters: PlugX, ShadowPad and Cobalt Strike attributed to Chinese threat actors; Remcos RAT attributed to a suspected India-linked group tracked as TAG-179. Named victims: Balochistan Police, Khyber Pakhtunkhwa Police, Islamabad Police, Punjab Safe Cities Authority (PSCA). Targeted assets include web applications holding biometric records, criminal case files, hotel/tenant registrations tied to national ID, and personnel records. One China-linked cluster planted a custom implant masquerading as a portal update. Motivational attribution: CPEC / Chinese-national security concerns for the China-nexus operator; long-standing India-Pakistan intelligence rivalry for TAG-179. Sources: [SentinelOne](https://www.sentinelone.com/labs/one-target-china-india-espionage-converge-on-pakistani-law-enforcement/), [The Hacker News](https://thehackernews.com/2026/07/hackers-weaponize-balochistan-police.html), [The Record](https://therecord.media/china-india-ran-separate-spy-campaigns-against-same-police-force).
- **Zimbra Classic Web Client — critical stored-XSS on email open, reporting wave 11 Jul** — [HIGH, CVE pending] Zimbra patched in Collaboration Suite 10.1.19 (released 07 Jul). Crafted email → script execution in the recipient's session on open → mailbox contents, session tokens and account settings exposed. Reported to Zimbra by Google TAG — a historical Winter Vivern / APT29 target class. No CVE yet; no confirmed in-the-wild use in-window. Priority patch for mail-serving telcos and government tenants. Sources: [BleepingComputer](https://www.bleepingcomputer.com/news/security/zimbra-urges-customers-to-patch-critical-web-client-xss-flaw/), [The Hacker News](https://thehackernews.com/2026/07/critical-zimbra-flaw-could-let-crafted_0483473395.html), [Security Affairs](https://securityaffairs.com/195130/hacking/update-now-critical-zimbra-classic-web-client-flaw-could-expose-mailboxes.html), [SC Media](https://www.scworld.com/brief/zimbra-urges-patching-of-critical-xss-vulnerability-in-classic-web-client).
- **U-Boot — six FIT-signature bugs (BRLY-2026-037 … BRLY-2026-042)** — [HIGH, embedded / BMC blast radius] Binarly's write-up dropped 09 Jul; industry pickup wave (BleepingComputer, The Hacker News, gbhackers, Cybersecurity News) rolled through 11 Jul. Two bugs allow arbitrary code execution during firmware signature verification (memory corruption in the FIT parser); four are DoS (OOB read, null-deref, malformed external-data validation, unbounded recursion). Vulnerable code has been in place since U-Boot 2013.07 → 50+ affected releases across enterprise BMCs, networking gear, industrial systems and IoT. Patched upstream; downstream vendor uptake is now the long tail. Sources: [Binarly](https://www.binarly.io/blog/unfit-to-boot-breaking-u-boots-fit-signature-verification), [BleepingComputer](https://www.bleepingcomputer.com/news/security/new-u-boot-flaws-could-enable-stealthy-firmware-attacks/), [The Hacker News](https://thehackernews.com/2026/07/six-new-u-boot-flaws-could-let.html), [gbhackers](https://gbhackers.com/multiple-u-boot-vulnerabilities/).
- **PraisonAI — critical path-traversal and SQLi CVEs published 11 Jul (all pre-4.6.78)** — [HIGH] NVD entries dated 11 Jul: a CVSS 9.8 path-traversal that fails to validate file-path references in custom command templates → arbitrary out-of-workspace file read; CVSS 8.6 SQL injection in the PGVector and Cassandra knowledge-store `create_collection()` backends via caller-controlled `dimension` interpolated straight into DDL; CVSS 7.5 SSRF via DNS-rebinding / HTTP-redirect bypass in the Crawl4AI/Chromium backend; and a prompt-injection defence misconfiguration (block threshold defaults to CRITICAL, allowing HIGH threats through). Upgrade to 4.6.78+. Relevant to anyone running Praison-based agent stacks — including internal RAG or agentic workflow platforms. Source: [NVD (via aggregated CVE feeds)](https://cve.akaoma.com/latest-cves).
- **ImageMagick 7.1.2-26 — multiple use-after-free CVEs published 11 Jul** — [NOTABLE, wide reach] Use-after-free in `FormatMagickCaption` on failed memory allocation, and a heap use-after-free in the XMP-profile parser. DoS or potential code execution on image ingestion. Massive downstream fan-out via every CMS, thumbnailer, mail gateway, ticketing tool and image-processing pipeline that shells out to ImageMagick. Patch fleets that transform user-supplied media. Source: [NVD (CVE-2026-61861 and siblings)](https://cve.threatint.eu/CVE/CVE-2026-61861).
