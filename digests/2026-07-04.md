## Telecom

- **NetNut / Popa residential-proxy botnet dismantled (Google TAG, FBI, IRS-CI).**
  ~2M compromised smart TVs, Android streaming boxes, and off-brand IoT nodes,
  many shipped pre-infected via bundled SDKs. In one week in June, Google saw
  316 distinct threat clusters — password-spraying, ad fraud, account takeover,
  cyber-espionage — exit through NetNut nodes. Why it matters: this was the
  clean-IP infrastructure a lot of telco-adjacent fraud (SIM/eSIM abuse,
  A2P/OTP spoofing, subscriber-portal ATO) rode on. Expect brief migration
  churn to alternate proxy networks; also expect fresh subscriber-side IP
  reputation issues on the affected ISPs. Consumer-router / STB advisories
  likely to follow.
  [Google TIG blog / Hacker News](https://thehackernews.com/2026/07/google-disrupts-netnut-residential.html)
  · [SecurityWeek](https://www.securityweek.com/google-fbi-disrupt-netnut-residential-proxy-network-powered-by-millions-of-devices/)
  · [BleepingComputer](https://www.bleepingcomputer.com/news/security/netnut-proxy-network-disrupted-2-million-infected-devices-cut-off/)
  · 2026-07-03 · [HIGH]

- **FatFs (elm-chan) — 7 unpatched vulns, embedded telecom kit in scope.**
  runZero disclosed CVE-2026-6682 (FAT32 mount integer overflow → memory
  corruption/RCE, 7.6), CVE-2026-6687 (exFAT volume-label overflow, 7.6),
  CVE-2026-6683 (exFAT div-by-zero, brickable during update flows, 4.6),
  CVE-2026-6686 (post-EOF read leaks deleted content, 4.6), plus three more.
  Why it matters to telco: FatFs is the FAT/exFAT layer in ESP-IDF, STM32Cube,
  Zephyr, MicroPython, TizenRT, SWUpdate and similar — i.e. it's inside a lot
  of small-cell radios, network edge appliances, GSM/LTE test kit, and OEM
  telematics/CPE. Physical-access threat model plus a maintainer who hasn't
  answered runZero for months means this ships unfixed today.
  [runZero](https://www.runzero.com/blog/fatfs-bugs/)
  · [Risky Bulletin](https://news.risky.biz/risky-bulletin-fatfs-bugs-enable-physical-access-attacks-on-a-load-of-devices/)
  · 2026-07-03 · [HIGH]

- **Warlock ransomware / Orange Belgium — reminder of exposure.** No new news

## General high-severity

- **"Bad Epoll" — Linux kernel LPE (CVE-2026-46242).** Race-condition
  use-after-free in `ep_remove()` — `file->f_ep` cleared, then the code
  continues using the now-NULL `file`; concurrent refcount drop frees the
  eventpoll while the removal path still holds a ref. Ends in an
  attacker-controlled `kmem_cache_free` on freed memory → arbitrary kernel
  write → root. ~6-instruction race window, but the reported exploit widens
  and retries it up to ~99% reliability without crashing. Reachable from
  inside a Chrome renderer sandbox and on Android. Reported by Jaeyoung Chung
  via Google kernelCTF (0-day submission). Introduced with the same 2023
  commit that produced CVE-2026-43074.
  [Hacker News](https://news.cybertechworld.co.in/index.php/2026/07/03/new-bad-epoll-linux-kernel-flaw-lets-unprivileged-users-gain-root-hits-android/)
  · [PoC](https://github.com/J-jaeyoung/bad-epoll)
  · 2026-07-03 · [HIGH]

- **Avalon — new modular malware framework carrying "CrownX" ransomware.**
  Undocumented framework covering cred theft, lateral movement, RAT,
  recovery-disruption, and ransomware in one payload. Delivered via
  password-protected Proton Drive archive + ISO-embedded stub (bypasses
  email-layer detection). Ships an unusually broad AV/EDR evasion list —
  Defender, SentinelOne, CrowdStrike, Sophos, Elastic, FortiEDR, ESET, McAfee,
  Bitdefender. Sample uploaded to VirusTotal 2026-03-11 with zero detections
  at time of writing.
  [Hacker News](https://thehackernews.com/2026/07/new-avalon-malware-framework-packs.html)
  · 2026-07-03 · [HIGH]

- **FortiBleed → INC + Lynx confirmed as a ransomware pipeline.** SOCRadar and
  BleepingComputer connected the FortiBleed sniffer campaign (custom
  `FortiGate Sniffer` on ~430k FortiGate boxes; ~110M credentials in one
  exposed staging server) to active affiliate negotiation panels on **both**
  INC and Lynx — same operator identity worked both. Adjacent to this,
  CVE-2026-35616 in FortiClient EMS (CVSS 9.1) was used to drop EKZ Stealer
  against an energy/utilities target. Practical read: if you had a FortiGate
  in the exposed set at any point, treat downstream ransomware as pre-staged.
  [Hacker News](https://thehackernews.com/2026/07/fortibleed-credential-theft-linked-to.html)
  · [BleepingComputer](https://www.bleepingcomputer.com/news/security/fortibleed-credential-theft-campaign-linked-to-lynx-ransomware/)
  · Reported 2026-07-02 / picked up on 03 · [HIGH]

- **ARToken — EvilTokens-affiliate PhaaS panel for M365.** Cisco Talos writeup:
  React-based operator console with 80+ API endpoints for OAuth device-code
  phishing, PRT persistence, mail read, BEC, and SharePoint exfil. Seven-layer
  anti-analysis chain (XOR-encrypted payloads + client-side behavioural
  checks). Underlying EvilTokens ecosystem tracked at ~500 Cloudflare-Workers
  domains and 2,000+ phishing pages; targeting finance/HR/logistics with
  AI-generated lures. Device-code phishing detections up 37× over baseline —
  now a commodity, not espionage-only.
  [Talos](https://blog.talosintelligence.com/artoken-inside-an-eviltokens-affiliate-panel-targeting-microsoft-365/)
  · [BleepingComputer](https://www.bleepingcomputer.com/news/security/artoken-phaas-exposes-eviltokens-microsoft-365-phishing-toolkit/)
  · Reported 2026-07-01 (still the dominant M365 hijack story in the window) · [NOTABLE]

- **Chromium browser-native ransomware POC.** Check Point flagged an
  AI-generated Flask app ("InfernoGrabber v9.0") that uses the File System
  Access API to enum, exfil, encrypt and overwrite local files — no native
  payload, no browser bug, no root. Works on Windows, macOS, Linux, Android,
  Edge; fails on iOS. Not seen in the wild yet, but the pattern will drift
  into commodity kits fast; worth pre-emptively reviewing FSA-API prompt
  UX and enterprise policy in browser MDM.
  [Hacker News](https://thehackernews.com/2026/07/ai-generated-browser-ransomware-abuses.html)
