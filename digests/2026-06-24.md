# Security digest — 2026-06-24

Window: 2026-06-23 04:00 UTC → 2026-06-24 04:00 UTC. The headline item is **active in-the-wild exploitation of Cisco Unified CM CVE-2026-20230** (SSRF → root), squarely in scope for enterprise voice and operator-managed UC. Beyond that: World Leaks dumped 630 GB stolen from **Tata Electronics**, Calif Security and the "Mythos" research preview disclosed **Squidbleed (CVE-2026-47729)**, and **FortiBleed** continues to expand — Russian-speaking IAB "SantaAd" now linked to a 110M-credential harvesting operation and a NATO defense-contractor compromise. **LastPass** confirmed its slice of the **Klue OAuth-token** supply-chain incident over the weekend. ## TL;DR

- **Cisco Unified CM CVE-2026-20230 (SSRF → root) under active exploitation** — Defused saw weekend probes from a single IP using `file://` payloads to write files on the device; Cisco patched on 2026-06-03 but WebDialer-enabled CUCM clusters that were not on 14SU6 (or interim COP) were exploitable. Public PoC code is out. WebDialer is off by default; many UC integrations turn it on. Not yet KEV-listed at write-time.
- **Tata Electronics — 630 GB / 200k+ files leaked by World Leaks** — Confirmed by Tata 2026-06-22/23; ransom group dumped emails, employee passport scans, multi-year event logs, and component / manufacturing design specs allegedly tied to Apple and Tesla. Tata says operations unaffected. World Leaks is the rebranded Hunters International / previously linked to the Nike breach.
- **Squidbleed (CVE-2026-47729) — 29-year-old heap over-read in Squid Proxy leaks cleartext HTTP requests** — Heartbleed-style cross-user leakage of `Authorization` headers / API keys. Squid 7.6 did NOT actually ship the fix despite earlier release notes; real patch lands in 7.7. Mostly impacts TLS-terminating or pure-HTTP setups.
- **FortiBleed expanded — Russian-speaking IAB "SantaAd" attributed, 110M creds harvested since Feb, NATO defense-contractor hit** — SecurityWeek, Recorded Future, SOCRadar, Field Effect, Hudson Rock, Bitsight, Arctic Wolf all corroborate. Cracked Kerberos hashes used on 2026-06-15 to exfil DFS backups from the contractor. Likely state-adjacent.
- **LastPass confirms Klue OAuth supply-chain breach exposed Salesforce CRM data** — Names, phones, emails, addresses, support-ticket contents. Vaults / product unaffected. Same Icarus-claimed incident also hit Recorded Future, Tanium, Jamf, Sprout Social, Gong, Insurity.

## Telecom

- **Cisco Unified CM CVE-2026-20230 — unauth SSRF → root, active exploitation observed** *(BleepingComputer 2026-06-23; Defused telemetry over the 2026-06-20/21 weekend)* — Unauthenticated remote attacker can send a crafted HTTP request to a CUCM or CUCM-SME node with WebDialer enabled; SSRF lets them write files to the host that later chain to root. CVSS 8.6 (Cisco PSIRT rates "Critical Security Impact"). Public PoC published the same week as the patch (2026-06-03). Defused saw `file://` payloads landing from a single IP on the weekend; broader scanning expected now that PoC is mature. Fixed in 14SU6 with interim COP; 15.x users must wait for 15SU5 (Sept 2026) or apply COP. For a telco offensive lens: CUCM is the dial-plan / call-control core for enterprise voice and most operator-managed UC tenants — root on a Publisher hands you the entire IP-PBX, voicemail, SIP-trunk credentials, and lawful-intercept hooks. Sweep every internet- or partner-reachable CUCM; verify WebDialer state; revoke any creds that touched the cluster post-2026-06-03. Not on KEV at write-time but a strong candidate for the next batch. ## General high-severity

- **Tata Electronics — World Leaks dumps 630 GB / 200k+ files** *(TechCrunch 2026-06-22; BleepingComputer / MacDailyNews / Security Boulevard 2026-06-23)* — Tata confirmed the breach to BleepingComputer this week; the actor is World Leaks (rebranded Hunters International, also tied to the Nike breach). Leaked dataset reportedly includes corporate email, multi-year event logs, employee passport scans (including foreign nationals), and a wide spread of manufacturing / component design specs from Tata's Apple and Tesla supplier relationships. Tata manufactures roughly one-third of Apple's iPhone output in India, so the supplier-secret blast radius is non-trivial; group is threatening further drops absent negotiation. No ransom paid; operations not impacted per Tata. [TechCrunch](https://techcrunch.com/2026/06/22/tata-electronics-a-major-tech-supplier-to-apple-and-tesla-confirms-data-breach/) · [BleepingComputer](https://www.bleepingcomputer.com/news/security/tata-electronics-confirms-cyberattack-as-hackers-leak-data/) · [MacDailyNews](https://macdailynews.com/2026/06/23/tata-electronics-confirms-cyber-breach-ransomware-group-leaks-alleged-apple-and-tesla-trade-secrets/) · **CRITICAL** (data exposure scale)
- **Squidbleed (CVE-2026-47729) — 29-year-old heap over-read in Squid Proxy** *(Calif Security disclosure 2026-06-22/23; The Register 2026-06-23)* — Cross-user memory leak in Squid's default config, regression dating to a January 1997 commit (missed null-terminator check before `strchr`). Bug walks past the buffer terminator and ships chunks of another user's recent HTTP request — typically `Authorization` headers, session tokens, or API keys — to a controlled FTP endpoint. Attack pre-requisites: cleartext HTTP traffic (not the CONNECT-tunneled HTTPS majority) and outbound TCP/21 to attacker. Squid 7.6 was *not* actually patched (despite earlier maintainer notes); fix is in 7.7. Calif Security credits "Claude Mythos Preview" as collaborator. Operators running Squid as a transparent forward proxy for management / OAM networks: audit your TLS-termination paths and check whether 7.6 is on your "patched" list. [Calif blog](https://blog.calif.io/p/squidbleed-cve-2026-47729) · [The Register](https://www.theregister.com/security/2026/06/23/mythos-discovers-squidbleed-a-memory-leak-thats-gone-undetected-since-clinton-era/5260367) · [The Hacker News](https://thehackernews.com/2026/06/29-year-old-squid-proxy-bug-squidbleed.html) · [SecurityAffairs](https://securityaffairs.com/194041/hacking/squidbleed-29-year-old-squid-bug-leaks-user-credentials.html) · **HIGH**
- **FortiBleed scoreboard — 430k FortiGates targeted, 110M creds harvested, NATO defense-contractor breached** *(SecurityWeek / Recorded Future analysis surfacing 2026-06-22/23, The Hacker News 2026-06-23)* — Russian-speaking IAB tracked as "SantaAd" attributed by SOCRadar/Recorded Future/Field Effect; tooling comments in Cyrillic. Custom sniffer turned compromised FortiGates into authentication-traffic collectors since Feb 2026, yielding ~110M raw credentials; verified working subset still ~86,644 across 194 countries. On 2026-06-15 SantaAd cracked Kerberos hashes offline and exfiltrated DFS backups from a NATO-aligned defense contractor — first concrete state-adjacent payoff observed. SMB-heavy victimology, US and India leading. No new CISA bulletin in 24h window, but the running playbook (kill sessions, rotate, phishing-resistant MFA, hunt for the FortigateSniffer / FortiBleed artifacts) is unchanged from CISA's 2026-06-18 guidance. [SecurityWeek (attribution)](https://www.securityweek.com/russian-initial-access-broker-behind-fortibleed-campaign/) · [Recorded Future](https://www.recordedfuture.com/blog/critical-fortibleed-campaign) · [The Hacker News (110M creds)](https://thehackernews.com/2026/06/fortibleed-targeted-fortigate-firewalls.html) · [SOCRadar whitepaper](https://socradar.io/resources/whitepapers/dismantling-fortibleed-inside-a-russian-fortinet-compromise-operation/) · **CRITICAL** (ongoing)
