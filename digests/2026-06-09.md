# Security digest — 2026-06-09

Window: last 24 hours ending 2026-06-09 ~04:00 UTC.

## TL;DR

- **Check Point Remote Access / Mobile Access VPN CVE-2026-50751 is the day's headline.** Critical (CVSS 9.3) authentication bypass in the deprecated IKEv1 key exchange; hotfix and emergency advisory landed in-window (2026-06-08). Active exploitation in the wild since **2026-05-07** — a month-long head start with no patch — and one cluster attributed with medium confidence to a **Qilin ransomware affiliate**. The same advisory ships **CVE-2026-50752** (CVSS 7.4, MitM against site-to-site VPN cert validation) for R81.20 / R82 / R82.10, no observed exploitation. If the perimeter still speaks IKEv1, treat as compromise-presumed and rotate VPN credentials.
- **CISA KEV adds two in-window: CVE-2026-50751 (Check Point) and CVE-2026-42271 (BerriAI LiteLLM).** LiteLLM is a command-injection / sub-process spawn via the MCP-preview endpoints (`/mcp-rest/test/connection`, `/mcp-rest/test/tools/list`); when paired with the Starlette auth-bypass CVE-2026-48710 it becomes unauthenticated RCE against the AI gateway. Fix in LiteLLM 1.83.7. FCEB deadline kicks the federal patch window for both.
- **Meta posts official notifications to 20,225 hijacked Instagram accounts.** Meta confirmed the in-window scale of the **High Touch Support (HTS)** AI password-reset attack: a code-path bug in HTS skipped the check that the requester's email matched the account's, so any user who told the chatbot they owned the target — from a VPN geo-located near the target — got the account linked to their address and the password reset. Accounts hit include the Obama-era White House handle and the US Space Force CMSgt. HTS disabled, all generated reset links revoked, mandatory security checkpoint enforced. Telco-adjacent because the carrier-level SMS OTP fallback would have blocked the original vector if 2FA had been on.
- **Linux nf_tables CVE-2026-23111 — Exodus Intelligence full writeup + working exploit drop in-window.** One inverted `!` in `nft_map_catchall_activate()` → UAF → unprivileged local root + container escape on Debian Bookworm/Trixie and Ubuntu 22.04/24.04 LTS. Upstream fix shipped 2026-02-05; FuzzingLabs PoC April; Exodus published the polished >99 % reliability exploit and full chain on 2026-06-08. Patch lag here is the story — anything still on a February-or-older kernel is now trivially rootable.
- **Microsoft Patch Tuesday lands today (2026-06-09 18:00 UTC).** Carry-over from yesterday's digest: Exchange Server CVE-2026-42897 (Critical, exploited) + the final Secure Boot dbx + cert window before 2026-06-26 hard cutoff. Deploy staged, not blanket.

## General high-severity

- **CISA KEV — two additions in-window (2026-06-08).** Both with FCEB remediation clocks now ticking.
  - **CVE-2026-50751 — Check Point Quantum Gateway / Mobile Access / Remote Access IKEv1 auth bypass.** Details above. CVSS 9.3, active exploitation since early May, Qilin ransomware affiliate linkage.
  - **CVE-2026-42271 — BerriAI LiteLLM command injection / RCE.** MCP-preview endpoints `/mcp-rest/test/connection` and `/mcp-rest/test/tools/list` accept full stdio-transport server configs (`command`, `args`, `env`) and spawn the subprocess. Any authenticated proxy-API-key holder gets command execution as the proxy process; **chained with Starlette CVE-2026-48710 (auth bypass), this becomes unauthenticated RCE** for any deployment whose dependency tree pulls Starlette ≤ 1.0.0. Affects LiteLLM 1.74.2 ≤ v < 1.83.7. Fix: 1.83.7. Public Horizon3 chain-PoC. **Why a practitioner cares:** LiteLLM is the AI-gateway most enterprises stand up to centralise model traffic and apply spend caps — it sits in front of OpenAI / Anthropic / Bedrock keys, so RCE on the gateway = exfil of every back-end API key the proxy holds. [Help Net Security](https://www.helpnetsecurity.com/2026/06/08/check-point-cve-2026-50751-qilin-ransomware/) · [The Hacker News (LiteLLM)](https://thehackernews.com/2026/06/critical-check-point-vpn-flaw-exploited.html) · [Horizon3.ai writeup](https://horizon3.ai/attack-research/vulnerabilities/cve-2026-42271-chained-with-cve-2026-48710/) · [runZero detection guide](https://www.runzero.com/blog/litellm/) · [NVD CVE-2026-42271](https://nvd.nist.gov/vuln/detail/CVE-2026-42271). Tag: **CRITICAL**.
- **Linux kernel CVE-2026-23111 — `nf_tables` UAF → local root + container escape, working exploit public.** Exodus Intelligence (Oliver Sieber) published the full Off-By-`!` writeup and weaponised exploit at >99 % reliability on idle systems on 2026-06-08. Root cause: a misplaced `!` in `nft_map_catchall_activate()` skips inactive catchall elements during the abort path → reactivation invariant breaks → UAF. Patched upstream 2026-02-05; FuzzingLabs published a PoC 2026-04-16. **Why this hurts now:** four months of patch lag means the population of vulnerable production kernels is large, and "container escape" makes this the LPE of choice against multi-tenant Kubernetes nodes. Affects Debian Bookworm / Trixie and Ubuntu 22.04 / 24.04 LTS in default configs. Distro patches are live (e.g. RHSA-2026:10996) — make sure managed-K8s nodes actually rebooted into the fixed kernel; container-only kernel patching does nothing for the host. [Exodus blog](https://blog.exodusintel.com/2026/06/08/off-by-exploiting-a-use-after-free-in-the-linux-kernel/) · [The Hacker News](https://thehackernews.com/2026/06/one-character-linux-kernel-flaw-enables.html) · [Cybersecurity News](https://cybersecuritynews.com/linux-kernel-nftables-vulnerability/) · [Red Hat RHSA-2026:10996](https://access.redhat.com/errata/RHSA-2026:10996) · [Ubuntu CVE tracker](https://ubuntu.com/security/CVE-2026-23111). Tag: **HIGH** (LPE, but mass-applicable).

## Worth watching

- **OpenAI ChatGPT "Lockdown Mode" rolled out in-window.** Optional account/workspace setting that sandboxes the model, blocks outbound URL-based exfil, and disables Deep Research / Agent Mode / live browsing — i.e., explicitly closes the prompt-injection exfil paths that several researchers have demonstrated against the agentic stack. Available on Free / Go / Plus / Pro and self-serve Business. Notable concession: OpenAI says it "substantially reduces" but **does not eliminate** prompt-injection exfil risk; memory / file uploads / model-training opt-in are unchanged. Anyone running ChatGPT against sensitive corpora through the standard product (not API) should flip it on for that workspace. [Help Net Security](https://www.helpnetsecurity.com/2026/06/08/openai-lockdown-mode-available/) · [Infosecurity Magazine](https://www.infosecurity-magazine.com/news/chatgpt-lockdown-mode-active/) · [The Cyber Express analysis](https://thecyberexpress.com/lockdown-mode-means-default-chatgpt-wasnt-safe/). - **VS Code 1.123 adds 2-hour auto-update delay for marketplace extensions.** Direct response to the May TanStack supply-chain incident (and a year of poisoned-extension reports).
